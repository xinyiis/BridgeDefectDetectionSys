## 1. 模块概述

### 1.1 功能定位

无人机管理模块负责管理用户的无人机设备信息，为视频流识别功能提供设备管理支持。是后续视频流检测任务的基础数据支撑。

### 1.2 核心功能

- **无人机列表查询**: 支持分页、权限过滤
- **无人机详情查看**: 查看设备基本信息和视频流配置
- **无人机添加**: 填写设备信息、配置视频流地址
- **无人机信息修改**: 更新设备信息、修改视频流地址
- **无人机删除**: 物理删除（无级联删除）

### 1.3 权限规则

| 角色 | 查看 | 添加 | 修改 | 删除 |
|------|------|------|------|------|
| 普通用户 | 仅自己的 | ✅ | 仅自己的 | 仅自己的 |
| 管理员 | 全部 | ✅ | 全部 | 全部 |

### 1.4 与其他模块的关系

```
┌──────────────┐
│  用户模块     │ (已完成)
└──────┬───────┘
       │ 依赖关系
       ↓
┌──────────────┐
│  无人机模块   │ (当前开发)
└──────┬───────┘
       │ 被依赖
       ↓
┌──────────────┐
│  检测模块     │ (后续开发 - 视频流识别)
└──────────────┘
```

**说明**:
- 无人机模块不直接依赖桥梁模块
- 但实现方式参考桥梁模块的权限控制设计
- 检测模块会关联无人机和桥梁

## 2. 功能需求分析

### 2.1 功能点详细分解

#### 2.1.1 查看无人机信息

**场景描述**: 用户需要查看自己的无人机设备，管理员需要查看所有无人机

**功能要求**:
- 列表展示（支持分页）
- 根据用户权限自动过滤数据
- 显示无人机基本信息（名称、型号、视频流地址）
- 支持查看单个无人机的详细信息

**数据来源**:
- 无人机基础信息 → `drones`表

**前端展示**:
- 无人机列表页: 卡片式展示，显示名称、型号
- 无人机详情页: 展示完整信息，包含视频流预览（可选）

#### 2.1.2 添加无人机

**场景描述**: 用户需要添加新的无人机设备

**功能要求**:
- 填写无人机名称（必填）
- 填写设备型号（可选）
- 配置视频流地址（可选，支持RTSP/RTMP/本地视频路径）
- 自动关联当前登录用户

**业务规则**:
- 名称长度: 1-100字符
- 型号长度: 最长100字符
- 视频流地址长度: 最长255字符
- 无唯一约束（允许同名同型号）

**错误处理**:
- 名称为空 → 400 Bad Request
- 名称超长 → 400 Bad Request
- 数据库插入失败 → 500 Internal Server Error

#### 2.1.3 修改无人机信息

**场景描述**: 用户需要更新无人机设备信息或修改视频流地址

**功能要求**:
- 修改无人机名称
- 修改设备型号
- 更新视频流地址
- 仅允许修改自己的无人机（管理员可修改所有）

**业务规则**:
- 所有字段都是可选的（至少修改一个字段）
- 权限验证: 普通用户只能修改自己的无人机
- 自动更新`updated_at`字段

#### 2.1.4 删除无人机

**场景描述**: 用户需要删除不再使用的无人机设备

**功能要求**:
- 物理删除（直接从数据库删除记录）
- 仅允许删除自己的无人机（管理员可删除所有）
- 无级联删除（缺陷记录不关联无人机）

**业务规则**:
- 权限验证: 普通用户只能删除自己的无人机
- 使用`Unscoped().Delete()`进行物理删除

**为什么物理删除？**
- 无人机数据简单，删除后无需恢复
- 无人机删除不涉及级联数据（缺陷记录关联的是桥梁，不是无人机）
- 简化实现，提高性能

```
    ├── drone_dto.go           # 数据传输对象
```

## 5. 数据模型设计

### 5.1 Drone 实体

```go
// Drone 无人机实体
// 对应数据库 drones 表
type Drone struct {
    ID        uint      `gorm:"primaryKey;autoIncrement" json:"id"`              // 无人机ID（主键）
    Name      string    `gorm:"type:varchar(100);not null" json:"name"`         // 无人机名称
    Model     string    `gorm:"type:varchar(100)" json:"model"`                 // 设备型号
    StreamURL string    `gorm:"type:varchar(255)" json:"stream_url"`            // 视频流地址
    UserID    uint      `gorm:"index;not null" json:"user_id"`                  // 所属用户ID（外键）
    User      *User     `gorm:"foreignKey:UserID" json:"user,omitempty"`        // 关联用户（延迟加载）
    CreatedAt time.Time `gorm:"autoCreateTime" json:"created_at"`               // 创建时间
    UpdatedAt time.Time `gorm:"autoUpdateTime" json:"updated_at"`               // 更新时间
}
```

### 5.2 字段说明

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | uint | 主键ID | 自增 |
| name | string | 无人机名称 | 非空，最长100字符 |
| model | string | 设备型号 | 最长100字符 |
| stream_url | string | 视频流地址 | 最长255字符 |
| user_id | uint | 所属用户ID | 非空，索引 |
| created_at | time.Time | 创建时间 | 自动生成 |
| updated_at | time.Time | 更新时间 | 自动更新 |

### 5.3 数据库索引

```sql
CREATE TABLE `drones` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT,
  `name` varchar(100) NOT NULL COMMENT '无人机名称',
  `model` varchar(100) DEFAULT NULL COMMENT '设备型号',
  `stream_url` varchar(255) DEFAULT NULL COMMENT '视频流地址',
  `user_id` bigint unsigned NOT NULL COMMENT '所属用户ID',
  `created_at` datetime DEFAULT NULL COMMENT '创建时间',
  `updated_at` datetime DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='无人机设备表';
```

### 5.4 辅助方法

```go
// TableName 指定表名

    return "drones"
}

// IsOwnedBy 判断无人机是否属于指定用户
```

## 7. 业务逻辑设计

### 7.1 DroneService 核心方法

#### 7.1.1 ListDrones - 分页查询（带权限过滤）

```go
func (s *DroneService) ListDrones(currentUser *model.User, page, pageSize int) ([]model.Drone, int64, error) {
    // 1. 参数验证
    if page < 1 {
        page = 1
    }
    if pageSize < 1 || pageSize > 100 {
        pageSize = 10
    }

    // 2. 构建查询
    query := s.db.Model(&model.Drone{})

    // 3. 权限过滤（Service层）
    if !currentUser.IsAdmin() {
        query = query.Where("user_id = ?", currentUser.ID)
    }

    // 4. 统计总数
    var total int64
    query.Count(&total)

    // 5. 分页查询
    var drones []model.Drone
    offset := (page - 1) * pageSize
    err := query.Offset(offset).Limit(pageSize).Order("created_at DESC").Find(&drones).Error

    return drones, total, err
}
```

**对比桥梁模块**:
- ✅ 相同的权限过滤逻辑
- ✅ 相同的分页查询逻辑
- ❌ 无状态过滤（无人机没有status字段）

#### 7.1.2 CreateDrone - 创建无人机

```go
func (s *DroneService) CreateDrone(drone *model.Drone) error {
    // 1. 参数验证
    if drone.Name == "" {
        return errors.New("无人机名称不能为空")
    }
    if len(drone.Name) > 100 {
        return errors.New("无人机名称不能超过100字符")
    }

    // 2. 直接插入数据库（无需检查唯一性）
    return s.db.Create(drone).Error
}
```

**对比桥梁模块**:
- ❌ 无需检查唯一约束（桥梁需要检查bridge_code）
- ❌ 无文件上传处理
- ✅ 更简单直接

#### 7.1.3 UpdateDrone - 更新无人机

```go
func (s *DroneService) UpdateDrone(drone *model.Drone) error {
    // 1. 参数验证
    if drone.Name != "" && len(drone.Name) > 100 {
        return errors.New("无人机名称不能超过100字符")
    }

    // 2. 直接更新
    return s.db.Model(drone).Updates(drone).Error
}
```

**对比桥梁模块**:
- ✅ 逻辑完全相同
- ❌ 无文件更新处理

#### 7.1.4 DeleteDrone - 删除无人机

```go
func (s *DroneService) DeleteDrone(droneID uint, currentUser *model.User) error {
    // 1. 查询无人机
    drone, err := s.droneRepo.FindByID(droneID)
    if err != nil {
        return err
    }

    // 2. 权限验证
    if !currentUser.IsAdmin() && !drone.IsOwnedBy(currentUser.ID) {
        return errors.New("权限不足")
    }

    // 3. 物理删除
    return s.db.Unscoped().Delete(&model.Drone{}, droneID).Error
}
```

**对比桥梁模块**:
- ✅ 相同的权限验证逻辑
- ❌ 物理删除（桥梁是软删除）
- ❌ 无级联删除（桥梁需要级联删除缺陷记录）
- ❌ 无文件删除（桥梁需要删除3D模型文件）

## 8. 权限控制设计

### 8.1 Service层权限过滤

**ListDrones - 数据库层面过滤**:
```go
if !currentUser.IsAdmin() {
    query = query.Where("user_id = ?", currentUser.ID)
}
```

- 普通用户: 只查询自己的无人机
- 管理员: 查询所有无人机
- 性能最优: 数据库层面过滤

### 8.2 中间件权限验证

**DroneOwnershipRequired - 资源所有权验证**:
```go
func DroneOwnershipRequired(droneRepo repository.DroneRepository) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. 获取当前用户
        currentUser := c.MustGet("user").(*model.User)

        // 2. 管理员直接放行
        if currentUser.IsAdmin() {
            c.Next()
            return
        }

        // 3. 获取无人机ID
        droneID := c.Param("id")

        // 4. 查询无人机
        drone, err := droneRepo.FindByID(droneID)
        if err != nil {
            response.NotFound(c, "无人机不存在")
            c.Abort()
            return
        }

        // 5. 验证所有权
        if drone.UserID != currentUser.ID {
            response.Forbidden(c, "无权访问此无人机")
            c.Abort()
            return
        }

        // 6. 存入上下文（避免Handler重复查询）
        c.Set("drone", drone)
        c.Next()
    }
}
```

**应用场景**:
- GET `/drones/:id` - 查看详情
- PUT `/drones/:id` - 更新信息
- DELETE `/drones/:id` - 删除无人机

**优化点**:
- 管理员直接放行（无需查询数据库）
- 查询结果存入Context（Handler可直接读取）

### 8.3 路由注册

```go
// 无人机管理
drones := r.Group("/drones")
{
    // 列表和创建不需要所有权验证
    drones.GET("", droneHandler.ListDrones)    // 获取列表
    drones.POST("", droneHandler.CreateDrone)  // 创建无人机

    // 单个资源操作需要所有权验证
    droneResource := drones.Group("/:id")
    droneResource.Use(middleware.DroneOwnershipRequired(droneRepo))
    {
        droneResource.GET("", droneHandler.GetDrone)       // 获取详情
        droneResource.PUT("", droneHandler.UpdateDrone)    // 更新信息
        droneResource.DELETE("", droneHandler.DeleteDrone) // 删除无人机
    }
}
```

## 总结

### 设计原则
1. **简单直接** - 无人机模块比桥梁模块简单很多
2. **复用架构** - 使用相同的DDD四层架构
3. **统一权限** - 使用相同的权限控制逻辑
4. **避免过度设计** - 不需要的功能不实现

### 开发优势
- ✅ 有桥梁模块经验，上手快
- ✅ 可复用中间件和工具函数
- ✅ 测试框架已搭建好
- ✅ 业务逻辑简单，不易出错

### 风险控制
- ⚠️ 物理删除需谨慎（无法恢复）
- ⚠️ 无唯一约束可能导致重复数据（业务可接受）
