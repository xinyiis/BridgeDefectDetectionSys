## 模块概述

### 功能定位
无人机管理模块负责管理用户的无人机设备信息，为视频流识别功能提供设备管理支持。

### 核心功能
1. **无人机CRUD** - 创建、查询、更新、删除无人机
2. **权限隔离** - 用户只能管理自己的无人机，管理员可以管理所有
3. **视频流地址管理** - 存储和管理无人机的视频流地址

### 业务场景
- 用户添加无人机设备，配置视频流地址
- 视频流识别时选择无人机设备
- 查看自己的无人机列表
- 管理员查看所有用户的无人机

## 数据模型设计

### Drone 实体

```go
type Drone struct {
    ID        uint      `gorm:"primaryKey;autoIncrement" json:"id"`              // 无人机ID（主键）
    Name      string    `gorm:"type:varchar(100);not null" json:"name"`         // 无人机名称
    Model     string    `gorm:"type:varchar(100)" json:"model"`                 // 设备型号
    StreamURL string    `gorm:"type:varchar(255)" json:"stream_url"`            // 视频流地址
    UserID    uint      `gorm:"index;not null" json:"user_id"`                  // 所属用户ID（外键）
    User      *User     `gorm:"foreignKey:UserID" json:"user,omitempty"`        // 关联用户（延迟加载）
    CreatedAt time.Time `gorm:"autoCreateTime" json:"created_at"`               // 创建时间
    UpdatedAt time.Time `gorm:"autoUpdateTime" json:"updated_at"`               // 更新时间
}
```

### 字段说明

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | uint | 主键ID | 自增 |
| name | string | 无人机名称 | 非空，最长100字符 |
| model | string | 设备型号 | 最长100字符 |
| stream_url | string | 视频流地址（RTSP/RTMP） | 最长255字符 |
| user_id | uint | 所属用户ID（外键） | 非空，索引 |
| created_at | time.Time | 创建时间 | 自动生成 |
| updated_at | time.Time | 更新时间 | 自动更新 |

### 设计说明

**与桥梁模块对比**：
- ✅ **更简单** - 无人机字段更少（8个 vs 桥梁18个）
- ✅ **无软删除** - 删除无人机直接物理删除（无级联数据）
- ✅ **无唯一约束** - 无人机名称可重复
- ✅ **无文件上传** - 无3D模型等文件

**为什么不需要软删除？**
- 无人机删除不涉及级联数据（缺陷记录关联的是桥梁，不是无人机）
- 无人机数据简单，删除后无需恢复

**为什么没有唯一约束？**
- 用户可能有多个同型号无人机
- 用名称区分即可，不需要强制唯一

## 业务逻辑设计

### DroneService 核心方法

#### 1. ListDrones - 分页查询（带权限过滤）

```go
func (s *DroneService) ListDrones(currentUser *model.User, page, pageSize int) ([]model.Drone, int64, error)
```

**实现逻辑**：
```go
query := s.db.Model(&model.Drone{})

// 权限过滤
if !currentUser.IsAdmin() {
    query = query.Where("user_id = ?", currentUser.ID)
}

// 分页查询
offset := (page - 1) * pageSize
query.Offset(offset).Limit(pageSize).Order("created_at DESC")
```

**对比桥梁模块**：
- ✅ 相同的权限过滤逻辑
- ✅ 相同的分页查询逻辑
- ❌ 无状态过滤（无人机没有status字段）

#### 4. DeleteDrone - 删除无人机

```go
func (s *DroneService) DeleteDrone(droneID uint, currentUser *model.User) error
```

**实现逻辑**：
```go
// 1. 查询无人机
drone, err := s.droneRepo.FindByID(droneID)

// 2. 权限验证
if !currentUser.IsAdmin() && !drone.IsOwnedBy(currentUser.ID) {
    return errors.New("权限不足")
}

// 3. 物理删除
return s.db.Unscoped().Delete(&model.Drone{}, droneID).Error
```

**对比桥梁模块**：
- ✅ 相同的权限验证逻辑
- ❌ 物理删除（桥梁是软删除）
- ❌ 无级联删除（桥梁需要级联删除缺陷记录）
- ❌ 无文件删除（桥梁需要删除3D模型文件）

## 权限控制

### 1. Service层权限过滤

**ListDrones - 数据库层面过滤**：
```go
if !currentUser.IsAdmin() {
    query = query.Where("user_id = ?", currentUser.ID)
}
```

- 普通用户：只查询自己的无人机
- 管理员：查询所有无人机
- 性能最优：数据库层面过滤

---

### 2. 中间件权限验证

**DroneOwnershipRequired - 资源所有权验证**：
```go
func DroneOwnershipRequired(droneRepo repository.DroneRepository) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. 获取当前用户
        currentUser := c.MustGet("user").(*model.User)

        // 2. 管理员直接放行
        if currentUser.IsAdmin() {
            c.Next()
            return
        }

        // 3. 获取无人机ID
        droneID := c.Param("id")

        // 4. 查询无人机
        drone, err := droneRepo.FindByID(droneID)

        // 5. 验证所有权
        if drone.UserID != currentUser.ID {
            response.Forbidden(c, "无权访问此无人机")
            c.Abort()
            return
        }

        // 6. 存入上下文（避免Handler重复查询）
        c.Set("drone", drone)
        c.Next()
    }
}
```

**应用场景**：
- GET /drones/:id - 查看详情
- PUT /drones/:id - 更新信息
- DELETE /drones/:id - 删除无人机

**优化点**：
- 管理员直接放行（无需查询数据库）
- 查询结果存入Context（Handler可直接读取）

3. ❌ 唯一索引释放（无唯一约束）
4. ❌ 级联删除（无关联数据）
5. ❌ 事务处理（单表操作）

**保留的功能**：
1. ✅ 权限自动过滤（Service层）
2. ✅ 资源所有权验证（中间件）
3. ✅ 上下文优化（避免重复查询）
4. ✅ 分页查询

### 5. 路由注册

```go
// 无人机管理
drones := r.Group("/drones")
{
    // 列表和创建不需要所有权验证
    drones.GET("", droneHandler.ListDrones)    // 获取列表

        droneResource.PUT("", droneHandler.UpdateDrone)    // 更新信息
        droneResource.DELETE("", droneHandler.DeleteDrone) // 删除无人机
    }
}
```

## 总结

### 设计原则
1. **简单直接** - 无人机模块比桥梁模块简单很多
2. **复用架构** - 使用相同的DDD四层架构
3. **统一权限** - 使用相同的权限控制逻辑
4. **避免过度设计** - 不需要的功能不实现
