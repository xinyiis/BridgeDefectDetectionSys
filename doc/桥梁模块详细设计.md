## 1. 模块概述

### 1.1 功能定位

桥梁管理模块是桥梁缺陷检测系统的核心业务模块，负责管理桥梁基础信息、3D模型文件和历史缺陷记录，为后续检测任务提供基础数据支撑。

### 1.2 核心功能

- **桥梁列表查询**: 支持分页、权限自动过滤
- **桥梁详情查看**: 包含基本信息、3D模型、历史缺陷
- **桥梁添加**: 填写基本信息、上传3D模型（可选）
- **桥梁信息修改**: 更新基本信息、替换3D模型
- **桥梁删除**: 软删除、级联删除关联缺陷记录

### 1.3 权限规则

| 角色 | 查看 | 添加 | 修改 | 删除 |
|------|------|------|------|------|
| 普通用户 | 仅自己的 | ✅ | 仅自己的 | 仅自己的 |
| 管理员 | 全部 | ✅ | 全部 | 全部 |

## 2. 核心改进点分析

### 2.1 与用户模块对比

| 对比维度 | 用户模块 | 桥梁模块 | 技术实现 | 难度 |
|---------|---------|---------|---------|------|
| **数据模型** | 简单（无软删除） | 完整（带软删除） | 添加`gorm.DeletedAt` | ⭐ |
| **权限控制** | Session + 角色 | Session + 角色 + **资源所有权** | 新增中间件验证 | ⭐⭐ |
| **文件处理** | 无 | 3D模型上传/存储/删除 | 文件服务 + multipart/form-data | ⭐⭐ |
| **关联数据** | 无 | 缺陷记录（一对多） | Preload预加载 | ⭐⭐ |
| **级联删除** | 无 | 软删除 + 级联 | GORM事务处理 | ⭐⭐⭐ |

### 2.2 技术挑战点

#### 挑战1: 软删除与唯一索引冲突 ⚠️

**问题**: 桥梁编号有唯一索引，软删除后无法创建相同编号的新桥梁

**影响**: 数据完整性约束

**解决方案**: 删除时在编号后加时间戳标记

#### 挑战2: 资源所有权验证 ⚠️

**问题**: 需要在访问资源前验证用户是否有权限

**影响**: 安全性

**解决方案**: 中间件查询数据库验证所有权

#### 挑战3: 文件上传失败处理 ⚠️

**问题**: 文件上传成功但数据库创建失败，产生孤立文件

**影响**: 存储空间浪费

**解决方案**: 失败时回滚，删除已上传文件

#### 挑战4: 级联删除数据一致性 ⚠️

**问题**: 删除桥梁时需要同时删除缺陷和文件

**影响**: 数据一致性

**解决方案**: 使用GORM事务保证原子性

```go
package model

import (

// Bridge 桥梁实体
type Bridge struct {
    ID          uint           `gorm:"primaryKey;autoIncrement" json:"id"`
    BridgeName  string         `gorm:"type:varchar(100);not null" json:"bridge_name"`
    BridgeCode  string         `gorm:"type:varchar(50);uniqueIndex;not null" json:"bridge_code"`
    Address     string         `gorm:"type:varchar(255)" json:"address"`
    Longitude   float64        `gorm:"type:decimal(10,6)" json:"longitude"`
    Latitude    float64        `gorm:"type:decimal(10,6)" json:"latitude"`
    BridgeType  string         `gorm:"type:varchar(50)" json:"bridge_type"`     // 梁桥/拱桥/斜拉桥
    BuildYear   int            `gorm:"type:int" json:"build_year"`
    Length      float64        `gorm:"type:decimal(10,2)" json:"length"`        // 米
    Width       float64        `gorm:"type:decimal(10,2)" json:"width"`         // 米
    Status      string         `gorm:"type:varchar(20);default:'正常'" json:"status"`
    Model3DPath string         `gorm:"type:varchar(255)" json:"model_3d_path"`  // 3D模型路径
    Remark      string         `gorm:"type:text" json:"remark"`
    UserID      uint           `gorm:"not null;index" json:"user_id"`           // 外键
    User        *User          `gorm:"foreignKey:UserID" json:"user,omitempty"` // 关联用户
    Defects     []Defect       `gorm:"foreignKey:BridgeID" json:"defects,omitempty"` // 关联缺陷
    CreatedAt   time.Time      `gorm:"autoCreateTime" json:"created_at"`
    UpdatedAt   time.Time      `gorm:"autoUpdateTime" json:"updated_at"`
    DeletedAt   gorm.DeletedAt `gorm:"index" json:"-"`                          // 🔑 软删除字段
}

// TableName 指定表名
func (Bridge) TableName() string {
    return "bridges"
}

// IsOwnedBy 判断桥梁是否属于指定用户
func (b *Bridge) IsOwnedBy(userID uint) bool {
    return b.UserID == userID
}
```

## 5. 核心技术实现方案

### 5.1 软删除实现 ⭐⭐⭐

#### 5.1.1 GORM软删除机制

**关键字段**：
```go
DeletedAt gorm.DeletedAt `gorm:"index" json:"-"`
```

**自动行为**：

| 操作 | SQL生成 | 说明 |
|------|---------|------|
| 删除 | `UPDATE bridges SET deleted_at = NOW() WHERE id = ?` | 软删除，不物理删除 |
| 查询 | `SELECT * FROM bridges WHERE deleted_at IS NULL` | 自动过滤已删除 |
| 查询已删除 | `db.Unscoped().Find(&bridges)` | 使用Unscoped查看 |

#### 5.1.2 软删除与唯一索引冲突处理

**问题场景**：
```
1. 创建桥梁（bridge_code='BR001'）
2. 软删除 → deleted_at='2026-03-15 19:00:00'
3. 再次创建（bridge_code='BR001'）
   ❌ 错误：Duplicate entry 'BR001' for key 'idx_bridge_code'
```

**解决方案：删除时修改编号**

```go
// internal/infrastructure/persistence/bridge_repository_impl.go
func (r *BridgeRepositoryImpl) Delete(id uint) error {
    return r.db.Transaction(func(tx *gorm.DB) error {
        var bridge model.Bridge
        if err := tx.First(&bridge, id).Error; err != nil {
            return err
        }

        // 在编号后加时间戳，释放唯一索引
        timestamp := time.Now().Unix()
        bridge.BridgeCode = fmt.Sprintf("%s_deleted_%d", bridge.BridgeCode, timestamp)

        if err := tx.Save(&bridge).Error; err != nil {
            return err
        }

        // 软删除
        return tx.Delete(&bridge).Error
    })
}
```

**效果**：
- 删除前：`bridge_code='BR001'`, `deleted_at=NULL`
- 删除后：`bridge_code='BR001_deleted_1710504000'`, `deleted_at='2026-03-15 19:00:00'`

#### 5.2.2 中间件实现

```go
// internal/interfaces/http/middleware/resource_ownership.go
package middleware

import (
    "strconv"
    "github.com/gin-gonic/gin"
    "github.com/xinyiis/BridgeDefectDetectionSys/src/backend/internal/domain/model"
    "github.com/xinyiis/BridgeDefectDetectionSys/src/backend/internal/domain/service"
    "github.com/xinyiis/BridgeDefectDetectionSys/src/backend/pkg/response"
)

// BridgeOwnershipRequired 验证用户是否有权访问桥梁资源
func BridgeOwnershipRequired(bridgeService *service.BridgeService) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. 获取当前用户
        currentUser, exists := c.Get("current_user")
        if !exists {
            response.Unauthorized(c)
            c.Abort()
            return
        }
        user := currentUser.(*model.User)

        // 2. 管理员直接放行（不查数据库）
        if user.IsAdmin() {
            c.Next()
            return
        }

        // 3. 获取桥梁ID
        bridgeIDStr := c.Param("id")
        bridgeID, err := strconv.ParseUint(bridgeIDStr, 10, 64)
        if err != nil {
            response.BadRequest(c, "无效的桥梁ID")
            c.Abort()
            return
        }

        // 4. 查询桥梁（验证存在性和所有权）
        bridge, err := bridgeService.GetByID(uint(bridgeID))
        if err != nil {
            response.NotFound(c, "桥梁不存在")
            c.Abort()
            return
        }

        // 5. 验证所有权
        if !bridge.IsOwnedBy(user.ID) {
            response.Forbidden(c)
            c.Abort()
            return
        }

        // 6. 将桥梁信息存入上下文（避免Handler重复查询）
        c.Set("bridge", bridge)
        c.Next()
    }
}
```

```go
func RegisterBridgeRoutes(r *gin.RouterGroup, handler *handler.BridgeHandler, bridgeService *service.BridgeService) {
    bridges := r.Group("/bridges")
    bridges.Use(middleware.AuthRequired())  // 🔑 认证中间件

            bridgeResource.GET("", handler.GetBridge)
            bridgeResource.PUT("", handler.UpdateBridge)
            bridgeResource.DELETE("", handler.DeleteBridge)
            bridgeResource.GET("/defects", handler.GetBridgeDefects)
        }
    }
}
```

#### 5.3.2 本地文件存储实现

```go
// internal/infrastructure/persistence/file_storage.go
package persistence

import (
    "fmt"
    "io"
    "os"
    "path/filepath"
    "mime/multipart"
    "github.com/google/uuid"
)

type LocalFileStorage struct {
    baseDir string  // 基础目录，如: "./uploads"
}

func NewLocalFileStorage(baseDir string) *LocalFileStorage {
    return &LocalFileStorage{baseDir: baseDir}
}

// SaveUploadedFile 保存上传文件
func (s *LocalFileStorage) SaveUploadedFile(file *multipart.FileHeader, dir string) (string, error) {
    // 1. 生成唯一文件名（UUID + 原扩展名）
    ext := filepath.Ext(file.Filename)
    filename := fmt.Sprintf("%s%s", uuid.New().String(), ext)

    // 2. 构建完整路径: uploads/models/uuid.obj
    fullPath := filepath.Join(s.baseDir, dir, filename)

    // 3. 确保目录存在
    if err := os.MkdirAll(filepath.Dir(fullPath), 0755); err != nil {
        return "", err
    }

    // 4. 打开上传文件
    src, err := file.Open()
    if err != nil {
        return "", err
    }
    defer src.Close()

    // 5. 创建目标文件
    dst, err := os.Create(fullPath)
    if err != nil {
        return "", err
    }
    defer dst.Close()

    // 6. 复制文件内容
    if _, err = io.Copy(dst, src); err != nil {
        return "", err
    }

    // 7. 返回相对路径（存入数据库）
    return filepath.Join(dir, filename), nil
}

// DeleteFile 删除文件
func (s *LocalFileStorage) DeleteFile(path string) error {
    fullPath := filepath.Join(s.baseDir, path)
    return os.Remove(fullPath)
}

// ValidateFileFormat 验证文件格式
func (s *LocalFileStorage) ValidateFileFormat(file *multipart.FileHeader, allowedExts []string) error {
    ext := filepath.Ext(file.Filename)
    for _, allowed := range allowedExts {
        if ext == allowed {
            return nil
        }
    }
    return fmt.Errorf("不支持的文件格式: %s，允许的格式: %v", ext, allowedExts)
}

// ValidateFileSize 验证文件大小
func (s *LocalFileStorage) ValidateFileSize(file *multipart.FileHeader, maxSize int64) error {
    if file.Size > maxSize {
        return fmt.Errorf("文件大小超过限制: %.2f MB (最大 %.2f MB)",
            float64(file.Size)/1024/1024,
            float64(maxSize)/1024/1024)
    }
    return nil
}
```

#### 5.3.3 Handler中使用文件服务

```go
// internal/interfaces/http/handler/bridge_handler.go
func (h *BridgeHandler) CreateBridge(c *gin.Context) {
    // 1. 绑定表单数据
    var req dto.CreateBridgeRequest
    if err := c.ShouldBind(&req); err != nil {
        response.BadRequest(c, "参数错误: "+err.Error())
        return
    }

    // 2. 获取当前用户
    currentUser, _ := c.Get("current_user")
    user := currentUser.(*model.User)

    // 3. 处理文件上传（可选）
    file, err := c.FormFile("model_3d_file")
    var model3DPath string

    if err == nil && file != nil {
        // 3.1 验证文件格式
        allowedFormats := []string{".obj", ".fbx", ".gltf", ".glb"}
        if err := h.fileService.ValidateFileFormat(file, allowedFormats); err != nil {
            response.BadRequest(c, err.Error())
            return
        }

        // 3.2 验证文件大小（50MB）
        if err := h.fileService.ValidateFileSize(file, 50*1024*1024); err != nil {
            response.BadRequest(c, err.Error())
            return
        }

        // 3.3 保存文件
        model3DPath, err = h.fileService.SaveUploadedFile(file, "models")
        if err != nil {
            response.InternalError(c, "文件上传失败")
            return
        }
    }

    // 4. 创建桥梁
    req.Model3DPath = model3DPath
    req.UserID = user.ID
    bridge, err := h.useCase.CreateBridge(c.Request.Context(), &req)

    if err != nil {
        // 🔑 关键：失败时删除已上传的文件
        if model3DPath != "" {
            h.fileService.DeleteFile(model3DPath)
        }
        response.BadRequest(c, err.Error())
        return
    }

    // 5. 返回结果
    response.Success(c, gin.H{
        "bridge_id":   bridge.ID,
        "bridge_name": bridge.BridgeName,
        "bridge_code": bridge.BridgeCode,
    })
}
```

#### 5.4.1 级联删除流程

```
┌─────────────────────┐
│ DELETE请求          │ DELETE /api/bridges/123
└──────┬──────────────┘
       │
```

```go
package service

import (
    "errors"
    "fmt"
    "github.com/xinyiis/BridgeDefectDetectionSys/src/backend/internal/domain/model"
    "github.com/xinyiis/BridgeDefectDetectionSys/src/backend/internal/domain/repository"
    "gorm.io/gorm"
)

type BridgeService struct {
    db          *gorm.DB
    bridgeRepo  repository.BridgeRepository
    fileService FileService
}

func NewBridgeService(db *gorm.DB, bridgeRepo repository.BridgeRepository, fileService FileService) *BridgeService {
    return &BridgeService{
        db:          db,
        bridgeRepo:  bridgeRepo,
        fileService: fileService,
    }
}

// DeleteBridge 删除桥梁（软删除 + 级联删除）
func (s *BridgeService) DeleteBridge(bridgeID uint, currentUser *model.User) error {
    // 1. 查询桥梁（验证存在性）
    bridge, err := s.bridgeRepo.FindByID(bridgeID)
    if err != nil {
        return errors.New("桥梁不存在")
    }

    // 2. 权限验证
    if !currentUser.IsAdmin() && !bridge.IsOwnedBy(currentUser.ID) {
        return errors.New("权限不足")
    }

    // 3. 开启事务（保证数据一致性）
    return s.db.Transaction(func(tx *gorm.DB) error {
        // 3.1 软删除桥梁
        if err := tx.Delete(&model.Bridge{}, bridgeID).Error; err != nil {
            return fmt.Errorf("删除桥梁失败: %w", err)
        }
        // 生成SQL: UPDATE bridges SET deleted_at = NOW() WHERE id = ?

        // 3.2 级联软删除所有关联的缺陷记录
        if err := tx.Where("bridge_id = ?", bridgeID).Delete(&model.Defect{}).Error; err != nil {
            return fmt.Errorf("删除缺陷记录失败: %w", err)
        }
        // 生成SQL: UPDATE defects SET deleted_at = NOW() WHERE bridge_id = ?

        // 3.3 删除3D模型文件（同步删除，不影响事务）
        if bridge.Model3DPath != "" {
            if err := s.fileService.DeleteFile(bridge.Model3DPath); err != nil {
                // 🔑 关键：文件删除失败只记录日志，不回滚事务
                // 文件可以后续通过清理任务删除
                log.Warnf("删除3D模型文件失败: %s, error: %v", bridge.Model3DPath, err)
            }
        }

        return nil
    })
}
```

## 7. 重点难点分析

### 7.1 唯一索引与软删除冲突

**难度**: ⭐⭐⭐

**问题描述**:
桥梁编号`bridge_code`有唯一索引，软删除后无法创建相同编号的新桥梁。

**解决方案**:

**方案1: 删除时修改编号（推荐）**
```go
func (r *BridgeRepositoryImpl) Delete(id uint) error {
    return r.db.Transaction(func(tx *gorm.DB) error {
        var bridge model.Bridge
        tx.First(&bridge, id)

        // 释放唯一索引
        timestamp := time.Now().Unix()
        bridge.BridgeCode = fmt.Sprintf("%s_deleted_%d", bridge.BridgeCode, timestamp)
        tx.Save(&bridge)

        return tx.Delete(&bridge).Error
    })
}
```

**方案2: 使用部分唯一索引（MySQL 8.0.13+）**
```sql
CREATE UNIQUE INDEX idx_bridge_code_active
ON bridges(bridge_code)
WHERE deleted_at IS NULL;
```

### 7.2 文件上传失败处理

**难度**: ⭐⭐

**问题描述**:
文件上传成功，但数据库创建失败，导致孤立文件。

**解决方案**: 失败时回滚，删除已上传文件

```go
func (h *BridgeHandler) CreateBridge(c *gin.Context) {
    // 1. 上传文件
    model3DPath, err := h.fileService.SaveUploadedFile(file, "models")

    // 2. 创建桥梁
    bridge, err := h.useCase.CreateBridge(req)
    if err != nil {
        // 🔑 关键：失败时删除已上传的文件
        if model3DPath != "" {
            h.fileService.DeleteFile(model3DPath)
        }
        response.BadRequest(c, err.Error())
        return
    }

    response.Success(c, bridge)
}
```

### 7.3 关联查询性能优化

**难度**: ⭐⭐

**问题描述**:
查询桥梁详情时需要加载缺陷列表，可能产生N+1查询。

**❌ 不好的做法**（N+1查询）:
```go
var bridges []model.Bridge
db.Find(&bridges)  // 查询100个桥梁

// 为每个桥梁查询缺陷（100次查询）
for i := range bridges {
    db.Where("bridge_id = ?", bridges[i].ID).Find(&bridges[i].Defects)
}
```

**✅ 推荐做法**（使用Preload）:
```go
// 一次性加载桥梁和缺陷（2次查询）
var bridges []model.Bridge
db.Preload("Defects").Find(&bridges)

// 生成SQL:
// SELECT * FROM bridges WHERE deleted_at IS NULL;
// SELECT * FROM defects WHERE bridge_id IN (1,2,3,...) AND deleted_at IS NULL;
```

```go
    // 限制每页最大数量，防止一次查询过多数据

        pageSize = 100
    }
    if pageSize < 1 {
        pageSize = 10
    }

```

#### 优化2: 使用Preload预加载关联数据

```go
// ✅ 推荐：使用Preload
db.Preload("Defects").Find(&bridges)

// ❌ 避免：循环查询（N+1问题）
for i := range bridges {
    db.Where("bridge_id = ?", bridges[i].ID).Find(&bridges[i].Defects)
}
```

#### 优化3: 按需加载关联数据

```go
// 列表查询不加载关联数据
func ListBridges() {

// 详情查询才加载关联数据
func GetBridge(id uint) {
    db.Preload("Defects").First(&bridge, id)
}
```

## 12. 总结

### 12.1 核心改进点

1. **软删除** → Bridge模型添加`gorm.DeletedAt`字段
2. **资源所有权验证** → 中间件验证 + Service层过滤
3. **文件上传** → 文件服务 + 失败回滚
4. **级联删除** → GORM事务 + 软删除缺陷
5. **权限过滤查询** → 根据角色动态添加条件

### 12.2 技术要点

- ✅ GORM软删除特性自动处理`deleted_at`
- ✅ 中间件验证资源所有权，避免重复查询
- ✅ 文件上传失败时清理已上传文件
- ✅ 使用事务保证级联删除数据一致性
- ✅ Preload预加载避免N+1查询问题

### 12.3 开发建议

- **保持一致性**: 沿用用户模块的DDD分层架构
- **增量改进**: 在已有基础上添加新功能
- **避免过度设计**: 文件存储用本地文件系统
- **关注测试**: 覆盖率≥85%，重点测试权限和级联
