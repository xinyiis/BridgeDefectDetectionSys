## 1. 模块概述

### 1.1 功能定位

桥梁管理模块是桥梁缺陷检测系统的核心业务模块，负责管理桥梁基础信息、3D模型文件和历史缺陷记录。是后续检测任务的基础数据支撑。

### 1.2 核心功能

- **桥梁列表查询**: 支持分页、权限过滤
- **桥梁详情查看**: 包含基本信息、3D模型、历史缺陷
- **桥梁添加**: 填写基本信息、上传3D模型（可选）
- **桥梁信息修改**: 更新基本信息、替换3D模型
- **桥梁删除**: 软删除、级联删除关联缺陷记录

### 1.3 权限规则

| 角色 | 查看 | 添加 | 修改 | 删除 |
|------|------|------|------|------|
| 普通用户 | 仅自己的 | ✅ | 仅自己的 | 仅自己的 |
| 管理员 | 全部 | ✅ | 全部 | 全部 |

### 1.4 与其他模块的关系

## 2. 功能需求分析

### 2.1 功能点详细分解

#### 2.1.1 查看桥梁信息

**场景描述**: 用户需要查看自己管理的桥梁，管理员需要查看所有桥梁

**功能要求**:
- 列表展示（支持分页）
- 根据用户权限自动过滤数据
- 显示桥梁基本信息（名称、编号、地址、状态等）
- 支持查看单个桥梁的详细信息
- 展示桥梁的3D模型（前端Three.js渲染）
- 展示该桥梁的历史缺陷记录

**数据来源**:
- 桥梁基础信息 → `bridges`表
- 3D模型文件 → 文件系统
- 历史缺陷记录 → `defects`表（关联查询）

#### 2.1.2 添加桥梁

**场景描述**: 用户添加新的桥梁进行管理

**功能要求**:
- 填写桥梁基本信息（必填）
  - 桥梁名称
  - 桥梁编号
  - 详细地址
  - 经纬度坐标
  - 桥梁类型
  - 建造年份
  - 桥梁长度/宽度
  - 备注信息
- 上传3D模型文件（可选）
  - 支持格式：.obj, .fbx, .gltf
  - 文件大小限制：最大50MB
- 自动关联到当前登录用户

**验证规则**:
- 桥梁编号唯一性检查
- 经纬度格式验证
- 文件格式和大小验证

#### 2.1.3 修改桥梁信息

**场景描述**: 用户需要更新桥梁的基本信息或3D模型

**功能要求**:
- 更新桥梁基本信息（任意字段）
- 更新或删除3D模型文件
- 只能修改自己的桥梁（管理员可修改所有）

**验证规则**:
- 权限验证（资源所有权检查）
- 桥梁编号唯一性检查（如果修改编号）
- 新文件的格式和大小验证

#### 2.1.4 删除桥梁

**场景描述**: 用户删除不再需要管理的桥梁

**功能要求**:
- 软删除桥梁记录（保留数据）
- 级联删除该桥梁下的所有缺陷记录
- 删除关联的3D模型文件
- 只能删除自己的桥梁（管理员可删除所有）

**安全措施**:
- 二次确认提示
- 权限验证
- 事务处理（保证数据一致性）

## 3. 开发难度评估

### 3.1 与用户模块对比

| 维度 | 用户模块 | 桥梁模块 | 难度变化 |
|------|----------|----------|----------|
| 基础CRUD | ✅ | ✅ | 相同 |
| 权限控制 | Session认证 + 角色验证 | 认证 + 角色 + 资源所有权 | ⬆️ 提升 |
| 数据验证 | 用户名、邮箱、密码 | 编号唯一性、坐标格式 | 相同 |
| 文件处理 | 无 | 3D模型上传/存储/删除 | ⬆️ 新增 |
| 关联数据 | 无 | 缺陷记录（一对多） | ⬆️ 新增 |
| 级联删除 | 无 | 软删除 + 级联 | ⬆️ 新增 |

**难度等级**: ⭐⭐⭐ (中等)

### 3.2 技术挑战点

#### 挑战1: 文件上传和存储
**难度**: ⭐⭐
**原因**:
- 需要处理大文件上传（最大50MB）
- 需要验证文件格式
- 需要管理文件生命周期（删除旧文件）

**解决方案**:
- 使用Gin的文件上传API
- 创建专门的文件服务层
- 使用UUID生成唯一文件名
- 在删除桥梁时同步删除文件

#### 挑战2: 权限过滤
**难度**: ⭐⭐
**原因**:
- 需要根据用户角色动态过滤数据
- 查询时需要自动添加user_id条件

**解决方案**:
- 在Service层实现权限过滤逻辑
- 管理员查询全部，普通用户添加user_id条件
- 资源访问时验证所有权

#### 挑战3: 级联删除
**难度**: ⭐⭐⭐
**原因**:
- 需要同时删除桥梁、缺陷、文件
- 需要保证事务一致性
- 软删除需要特殊处理

**解决方案**:
- 使用GORM事务
- 使用GORM的软删除特性
- 删除文件采用异步清理（可选）

#### 挑战4: 关联数据查询
**难度**: ⭐⭐
**原因**:
- 查询桥梁详情时需要加载缺陷列表
- 需要避免N+1查询问题

**解决方案**:
- 使用GORM的Preload预加载
- 对于大量数据使用分页加载
- 使用索引优化查询性能

## 5. 数据模型设计

### 5.1 Bridge实体模型

```go
// internal/domain/model/bridge.go
package model

import (
    "time"
    "gorm.io/gorm"
)

// Bridge 桥梁实体
type Bridge struct {
    ID          uint           `gorm:"primaryKey;autoIncrement" json:"id"`
    BridgeName  string         `gorm:"type:varchar(100);not null" json:"bridge_name"`
    BridgeCode  string         `gorm:"type:varchar(50);uniqueIndex;not null" json:"bridge_code"`
    Address     string         `gorm:"type:varchar(255)" json:"address"`
    Longitude   float64        `gorm:"type:decimal(10,6)" json:"longitude"`
    Latitude    float64        `gorm:"type:decimal(10,6)" json:"latitude"`
    BridgeType  string         `gorm:"type:varchar(50)" json:"bridge_type"`
    BuildYear   int            `gorm:"type:int" json:"build_year"`
    Length      float64        `gorm:"type:decimal(10,2)" json:"length"`
    Width       float64        `gorm:"type:decimal(10,2)" json:"width"`
    Status      string         `gorm:"type:varchar(20);default:'正常'" json:"status"`
    Model3DPath string         `gorm:"type:varchar(255)" json:"model_3d_path"`
    Remark      string         `gorm:"type:text" json:"remark"`
    UserID      uint           `gorm:"not null;index" json:"user_id"`
    User        User           `gorm:"foreignKey:UserID" json:"user,omitempty"`
    Defects     []Defect       `gorm:"foreignKey:BridgeID" json:"defects,omitempty"`
    CreatedAt   time.Time      `gorm:"autoCreateTime" json:"created_at"`
    UpdatedAt   time.Time      `gorm:"autoUpdateTime" json:"updated_at"`
    DeletedAt   gorm.DeletedAt `gorm:"index" json:"-"`
}

// TableName 指定表名
func (Bridge) TableName() string {
    return "bridges"
}

// IsOwnedBy 判断桥梁是否属于指定用户
func (b *Bridge) IsOwnedBy(userID uint) bool {
    return b.UserID == userID
}
```

### 5.2 数据库表结构

```sql
CREATE TABLE `bridges` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT COMMENT '桥梁ID',
  `bridge_name` varchar(100) NOT NULL COMMENT '桥梁名称',
  `bridge_code` varchar(50) NOT NULL COMMENT '桥梁编号',
  `address` varchar(255) DEFAULT NULL COMMENT '详细地址',
  `longitude` decimal(10,6) DEFAULT NULL COMMENT '经度',
  `latitude` decimal(10,6) DEFAULT NULL COMMENT '纬度',
  `bridge_type` varchar(50) DEFAULT NULL COMMENT '桥梁类型（梁桥/拱桥/斜拉桥等）',
  `build_year` int DEFAULT NULL COMMENT '建造年份',
  `length` decimal(10,2) DEFAULT NULL COMMENT '桥梁长度（米）',
  `width` decimal(10,2) DEFAULT NULL COMMENT '桥梁宽度（米）',
  `status` varchar(20) DEFAULT '正常' COMMENT '桥梁状态（正常/维修中/停用）',
  `model_3d_path` varchar(255) DEFAULT NULL COMMENT '3D模型文件路径',
  `remark` text COMMENT '备注信息',
  `user_id` bigint unsigned NOT NULL COMMENT '所属用户ID',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` datetime DEFAULT NULL COMMENT '软删除时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_bridge_code` (`bridge_code`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_deleted_at` (`deleted_at`),
  CONSTRAINT `fk_bridges_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='桥梁表';
```

**索引设计**:
- `PRIMARY KEY (id)`: 主键索引
- `UNIQUE KEY (bridge_code)`: 桥梁编号唯一索引
- `KEY (user_id)`: 用户ID索引，用于权限过滤查询
- `KEY (deleted_at)`: 软删除索引，GORM自动使用

**权限**:
- 普通用户: 只返回自己的桥梁
- 管理员: 返回所有桥梁

```
?page=1&page_size=10&status=正常
```

|------|------|------|--------|------|

```go
// internal/infrastructure/persistence/file_storage.go
package persistence

import (
    "fmt"
    "os"
    "path/filepath"
    "mime/multipart"
    "github.com/google/uuid"
)

type LocalFileStorage struct {
    baseDir string
}

func NewLocalFileStorage(baseDir string) *LocalFileStorage {
    return &LocalFileStorage{baseDir: baseDir}
}

// SaveUploadedFile 保存上传文件
func (s *LocalFileStorage) SaveUploadedFile(file *multipart.FileHeader, dir string) (string, error) {
    // 1. 生成唯一文件名
    filename := s.GenerateUniqueFileName(file.Filename)

    // 2. 构建完整路径
    fullPath := filepath.Join(s.baseDir, dir, filename)

    // 3. 确保目录存在
    if err := os.MkdirAll(filepath.Dir(fullPath), 0755); err != nil {
        return "", err
    }

    // 4. 保存文件
    src, err := file.Open()
    if err != nil {
        return "", err
    }
    defer src.Close()

    dst, err := os.Create(fullPath)
    if err != nil {
        return "", err
    }
    defer dst.Close()

    if _, err = io.Copy(dst, src); err != nil {
        return "", err
    }

    // 5. 返回相对路径
    return filepath.Join(dir, filename), nil
}

// DeleteFile 删除文件
func (s *LocalFileStorage) DeleteFile(path string) error {
    fullPath := filepath.Join(s.baseDir, path)
    return os.Remove(fullPath)
}

// GenerateUniqueFileName 生成唯一文件名
func (s *LocalFileStorage) GenerateUniqueFileName(originalName string) string {
    ext := filepath.Ext(originalName)
    return fmt.Sprintf("%s%s", uuid.New().String(), ext)
}

// ValidateFileFormat 验证文件格式
func (s *LocalFileStorage) ValidateFileFormat(file *multipart.FileHeader, allowedExts []string) error {
    ext := filepath.Ext(file.Filename)
    for _, allowed := range allowedExts {
        if ext == allowed {
            return nil
        }
    }
    return fmt.Errorf("不支持的文件格式: %s", ext)
}

// ValidateFileSize 验证文件大小
func (s *LocalFileStorage) ValidateFileSize(file *multipart.FileHeader, maxSize int64) error {
    if file.Size > maxSize {
        return fmt.Errorf("文件大小超过限制: %d MB", maxSize/1024/1024)
    }
    return nil
}
```

```go
// internal/interfaces/http/middleware/resource_ownership.go
package middleware

import (
    "strconv"
    "github.com/gin-gonic/gin"
    "github.com/xinyiis/BridgeDefectDetectionSys/internal/domain/model"
    "github.com/xinyiis/BridgeDefectDetectionSys/internal/domain/service"
    "github.com/xinyiis/BridgeDefectDetectionSys/pkg/response"
)

// BridgeOwnershipRequired 验证用户是否有权访问桥梁资源
func BridgeOwnershipRequired(bridgeService *service.BridgeService) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. 获取当前用户
        currentUser, exists := c.Get("current_user")
        if !exists {
            response.Unauthorized(c)
            c.Abort()
            return
        }
        user := currentUser.(*model.User)

        // 2. 管理员直接放行
        if user.IsAdmin() {
            c.Next()
            return
        }

        // 3. 获取桥梁ID
        bridgeIDStr := c.Param("id")
        bridgeID, err := strconv.ParseUint(bridgeIDStr, 10, 64)
        if err != nil {
            response.BadRequest(c, "无效的桥梁ID")
            c.Abort()
            return
        }

        // 4. 查询桥梁
        bridge, err := bridgeService.GetByID(uint(bridgeID))
        if err != nil {
            response.NotFound(c, "桥梁不存在")
            c.Abort()
            return
        }

        // 5. 验证所有权
        if !bridge.IsOwnedBy(user.ID) {
            response.Forbidden(c)
            c.Abort()
            return
        }

        // 6. 将桥梁信息存入上下文（避免重复查询）
        c.Set("bridge", bridge)
        c.Next()
    }
}
```

## 10. 级联删除设计

### 10.1 软删除配置

GORM的软删除特性：
- 使用`gorm.DeletedAt`字段
- `Delete()`操作自动设置`deleted_at`时间戳
- 查询时自动过滤已删除记录
- 可使用`Unscoped()`查询已删除记录

### 10.2 级联删除实现

```go
// internal/domain/service/bridge_service.go
func (s *BridgeService) DeleteBridge(bridgeID uint, currentUser *model.User) error {
    // 1. 查询桥梁（验证存在性）
    bridge, err := s.bridgeRepo.FindByID(bridgeID)
    if err != nil {
        return errors.New("桥梁不存在")
    }

    // 2. 权限验证
    if !currentUser.IsAdmin() && !bridge.IsOwnedBy(currentUser.ID) {
        return errors.New("权限不足")
    }

    // 3. 开启事务，保证数据一致性
    return s.db.Transaction(func(tx *gorm.DB) error {
        // 3.1 软删除桥梁
        if err := tx.Delete(&model.Bridge{}, bridgeID).Error; err != nil {
            return fmt.Errorf("删除桥梁失败: %w", err)
        }

        // 3.2 级联软删除所有关联的缺陷记录
        if err := tx.Where("bridge_id = ?", bridgeID).Delete(&model.Defect{}).Error; err != nil {
            return fmt.Errorf("删除缺陷记录失败: %w", err)
        }

        // 3.3 删除3D模型文件（可以异步执行，不影响事务）
        if bridge.Model3DPath != "" {
            // 方式1: 同步删除（推荐）
            if err := s.fileService.DeleteFile(bridge.Model3DPath); err != nil {
                // 文件删除失败不回滚事务，只记录日志
                log.Warnf("删除3D模型文件失败: %s, error: %v", bridge.Model3DPath, err)
            }

            // 方式2: 异步删除（可选）
            // go s.fileService.DeleteFile(bridge.Model3DPath)
        }

        return nil
    })
}
```

```go
func (s *BridgeService) RestoreBridge(bridgeID uint) error {

        }
```

```go
    }

    w := httptest.NewRecorder()
    router.ServeHTTP(w, req)

    // 3. 验证桥梁已软删除
    var deletedBridge model.Bridge
    err := db.First(&deletedBridge, bridge.ID).Error
    assert.Error(t, err) // 应该查询不到

    // 4. 验证缺陷记录也已软删除
    var defectCount int64
    db.Model(&model.Defect{}).Where("bridge_id = ?", bridge.ID).Count(&defectCount)
    assert.Equal(t, int64(0), defectCount)

    // 5. 验证使用Unscoped可以查到
    err = db.Unscoped().First(&deletedBridge, bridge.ID).Error
    assert.NoError(t, err)
    assert.NotNil(t, deletedBridge.DeletedAt)
}
```
