# 桥梁检测系统API接口文档

## 1. 接口概述

本接口文档基于桥梁检测系统的需求，设计了完整的RESTful API接口，支持用户管理、桥梁管理、无人机管理、图片上传识别、视频流识别、智能分析、主页信息统计和报表生成等功能。

## 2. 基础信息

- **API基础URL**: `/api/v1`
- **请求方式**: GET/POST/PUT/DELETE
- **数据格式**: JSON
- **认证方式**: Session + Cookie
- **前端配置**: 所有请求需设置 `credentials: 'include'` 以携带Cookie
- **响应格式**:
  ```json
  {
    "code": 200,
    "message": "成功",
    "data": { ... }
  }
  ```

## 3. 认证接口

### 3.1 用户注册
- **URL**: `/api/v1/auth/register`
- **方法**: POST
- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | username | string | 是 | 用户名 |
  | password | string | 是 | 密码 |
  | real_name | string | 是 | 真实姓名 |
  | phone | string | 否 | 手机号 |
  | email | string | 是 | 邮箱 |
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "注册成功",
    "data": {
      "user_id": 1,
      "username": "user1",
      "role": "user"
    }
  }
  ```

### 3.2 用户登录
- **URL**: `/api/v1/auth/login`
- **方法**: POST
- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | username | string | 是 | 用户名/邮箱 |
  | password | string | 是 | 密码 |
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "登录成功",
    "data": {
      "user": {
        "user_id": 1,
        "username": "user1",
        "real_name": "张三",
        "email": "user1@example.com",
        "role": "user"
      }
    }
  }
  ```
- **说明**:
  - Session自动通过Cookie管理，无需前端手动存储
  - 响应头包含 `Set-Cookie`，浏览器自动保存
  - 后续请求浏览器自动携带Cookie

### 3.3 用户登出
- **URL**: `/api/v1/auth/logout`
- **方法**: POST
- **参数**: 无
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "登出成功"
  }
  ```

## 6. 桥梁管理接口

- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |

- **返回值**:

  ```json
  {

    }
  }
  ```

### 6.3 添加桥梁
- **URL**: `/api/v1/bridges`
- **方法**: POST
- **Content-Type**: `multipart/form-data`（文件上传）
- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | bridge_name | string | 是 | 桥梁名称 |
  | bridge_code | string | 是 | 桥梁编号（唯一） |
  | address | string | 是 | 详细地址 |
  | longitude | float | 是 | 经度 |
  | latitude | float | 是 | 纬度 |
  | bridge_type | string | 是 | 桥梁类型 |
  | build_year | int | 是 | 建造年份 |
  | length | float | 是 | 桥梁长度（米） |
  | width | float | 是 | 桥梁宽度（米） |
  | status | string | 否 | 桥梁状态，默认"正常" |
  | model_3d_file | file | 否 | 3D模型文件（.obj/.fbx/.gltf/.glb，最大50MB） |
  | remark | string | 否 | 备注 |
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "添加成功",
    "data": {
      "id": 1,
      "bridge_name": "大桥1",
      "bridge_code": "BR001"
    }
  }
  ```

### 6.4 更新桥梁信息
- **URL**: `/api/v1/bridges/{id}`
- **方法**: PUT
- **Content-Type**: `application/json`
- **权限**: 用户只能更新自己的桥梁，管理员可以更新所有
- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | bridge_name | string | 否 | 桥梁名称 |
  | address | string | 否 | 详细地址 |
  | longitude | float | 否 | 经度 |
  | latitude | float | 否 | 纬度 |
  | bridge_type | string | 否 | 桥梁类型 |
  | build_year | int | 否 | 建造年份 |
  | length | float | 否 | 桥梁长度 |
  | width | float | 否 | 桥梁宽度 |
  | status | string | 否 | 桥梁状态 |
  | remark | string | 否 | 备注 |
- **说明**: 桥梁编号（bridge_code）不可修改，3D模型文件修改需单独接口
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "更新成功",
    "data": {
      "id": 1,
      "bridge_name": "大桥1"
    }
  }
  ```

### 6.5 删除桥梁
- **URL**: `/api/v1/bridges/{id}`
- **方法**: DELETE
- **参数**: 无
- **权限**: 用户只能删除自己的桥梁，管理员可以删除所有
- **说明**: 软删除，会级联删除关联的缺陷记录和3D模型文件
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "删除成功"
  }
  ```

## 11. 报表生成接口

### 11.1 生成检测报告
- **URL**: `/api/v1/reports`
- **方法**: POST
- **Content-Type**: `application/json`
- **权限**: 需要登录
- **参数**:
  | 字段名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | report_name | string | 是 | 报表名称（1-200字符） |
  | report_type | string | 是 | 报表类型：bridge_inspection（桥梁检测）/ defect_analysis（缺陷分析）/ health_comparison（健康对比） |
  | bridge_id | int | 条件必填 | 关联桥梁ID（bridge_inspection和defect_analysis必填） |
  | bridge_ids | []int | 条件必填 | 桥梁ID列表（health_comparison必填，至少2个） |
  | start_time | string | 是 | 报表开始时间（格式：2006-01-02） |
  | end_time | string | 是 | 报表结束时间（格式：2006-01-02） |
- **说明**:
  - 报表异步生成，接口立即返回status=generating
  - 前端需轮询查询报表详情接口获取生成状态
  - 生成时间约3-10秒
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "报表生成任务已创建",
    "data": {
      "id": 1,
      "report_name": "长江大桥检测报告",
      "report_type": "bridge_inspection",
      "user_id": 1,
      "bridge_id": 1,
      "bridge_name": "长江大桥",
      "start_time": "2026-02-01T00:00:00Z",
      "end_time": "2026-03-01T00:00:00Z",
      "status": "generating",
      "created_at": "2026-03-17T10:00:00Z"
    }
  }
  ```

### 11.3 获取报表详情
- **URL**: `/api/v1/reports/:id`
- **方法**: GET
- **权限**: 需要登录，且只能查看自己的报表（管理员可查看所有）
- **参数**: 无
- **说明**:
  - 前端轮询此接口获取生成状态
  - 建议轮询间隔3秒
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "成功",
    "data": {
      "id": 1,
      "report_name": "长江大桥2月检测报告",
      "report_type": "bridge_inspection",
      "user_id": 1,
      "bridge_id": 1,
      "bridge_name": "长江大桥",
      "start_time": "2026-02-01T00:00:00Z",
      "end_time": "2026-03-01T00:00:00Z",
      "file_path": "./reports/report_1_20260317100530.pdf",
      "file_size": 2048576,
      "status": "completed",
      "total_pages": 5,
      "defect_count": 15,
      "high_risk_count": 3,
      "health_score": 75.0,
      "created_at": "2026-03-17T10:00:00Z"
    }
  }
  ```
- **状态说明**:
  - `generating`: 生成中（轮询继续）
  - `completed`: 已完成（可以下载）
  - `failed`: 失败（查看error_message）

### 11.4 下载报表（新增）
- **URL**: `/api/v1/reports/:id/download`
- **方法**: GET
- **权限**: 需要登录，且只能下载自己的报表（管理员可下载所有）
- **参数**: 无
- **说明**:
  - 仅当status=completed时可下载
  - 浏览器自动下载PDF文件
- **返回值**: PDF文件流
- **响应头**:
  ```
  Content-Type: application/pdf
  Content-Disposition: attachment; filename=长江大桥2月检测报告.pdf
  ```

### 11.5 删除报表（新增）
- **URL**: `/api/v1/reports/:id`
- **方法**: DELETE
- **权限**: 需要登录，且只能删除自己的报表（管理员可删除所有）
- **参数**: 无
- **说明**:
  - 软删除（deleted_at标记）
  - 同时删除PDF文件
- **返回值**:
  ```json
  {
    "code": 200,
    "message": "删除成功"
  }
  ```
