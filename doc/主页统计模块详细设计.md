## 1. 模块概述

### 1.1 功能定位

主页信息统计模块是桥梁缺陷检测系统的**数据分析和可视化中心**，为用户提供系统核心数据的汇总、分析和趋势展示，帮助用户快速了解桥梁健康状况和系统使用情况。

### 1.2 核心价值

- **数据汇总**: 一屏展示所有关键指标
- **趋势分析**: 发现缺陷增长趋势和规律
- **决策支持**: 为桥梁维护计划提供数据依据
- **告警提示**: 高危缺陷和异常情况及时提醒

### 1.3 业务场景

#### 场景1：桥梁管理员每日巡检
**用户故事**：
> 作为桥梁管理员，我每天早上登录系统，首先查看主页统计，了解：
> - 昨天新增了多少缺陷？
> - 哪座桥梁缺陷最多，需要优先处理？
> - 最近一周缺陷趋势是上升还是下降？

**数据需求**：
- 基础统计卡片（总数 + 增量）
- 桥梁健康度排名
- 7天/30天缺陷趋势图

---

#### 场景2：系统管理员全局监控
**用户故事**：
> 作为系统管理员，我需要监控整个平台的运行情况：
> - 有多少活跃用户？
> - 系统中管理了多少座桥梁？
> - 哪种类型的缺陷最常见？

**数据需求**：
- 全局统计（所有用户数据聚合）
- 缺陷类型分布（饼图）
- 用户活跃度分析

---

#### 场景3：普通用户查看自己的数据
**用户故事**：
> 作为普通用户，我登录系统查看自己负责的桥梁：
> - 我管理了几座桥梁？
> - 这些桥梁总共有多少缺陷？
> - 最近一次检测是什么时候？

**数据需求**：
- 个人维度统计（仅当前用户的数据）
- 最近检测记录
- 待处理告警

## 2. 业务需求分析

### 2.1 核心统计指标

| 统计维度 | 指标名称 | 计算逻辑 | 权限过滤 |
|----------|----------|----------|----------|
| **基础统计** | 桥梁总数 | COUNT(bridges) | 用户只看自己的 |
| | 无人机总数 | COUNT(drones) | 用户只看自己的 |
| | 缺陷总数 | COUNT(defects) | JOIN过滤 |
| | 检测任务数 | COUNT(DISTINCT image_path) | JOIN过滤 |
| **增量统计** | 今日新增缺陷 | WHERE detected_at >= TODAY | JOIN过滤 |
| | 本周新增缺陷 | WHERE detected_at >= THIS_WEEK | JOIN过滤 |
| **缺陷分类** | 裂缝数量 | COUNT WHERE type='裂缝' | JOIN过滤 |
| | 剥落数量 | COUNT WHERE type='剥落' | JOIN过滤 |
| | 锈蚀数量 | COUNT WHERE type='锈蚀' | JOIN过滤 |
| **桥梁健康** | 桥梁缺陷排名 | GROUP BY bridge_id, ORDER BY count | JOIN过滤 |
| | 高危桥梁数 | WHERE defect_count > 阈值 | JOIN过滤 |

### 2.2 时间维度分析

| 时间范围 | 统计粒度 | 应用场景 |
|----------|----------|----------|
| 最近24小时 | 按小时 | 实时监控 |
| 最近7天 | 按天 | 周趋势分析 |
| 最近30天 | 按天 | 月度报告 |
| 最近12个月 | 按月 | 年度趋势 |
| 自定义时间范围 | 按天/周/月 | 报表生成 |

### 2.3 数据展示优先级

**优先级P0（必须实现）**：
1. ✅ 基础统计卡片（桥梁、无人机、缺陷、任务数）
2. ✅ 缺陷类型分布（饼图/柱状图）
3. ✅ 缺陷趋势图（7天/30天）
4. ✅ 桥梁健康度排名（Top 5/10）

**优先级P1（推荐实现）**：
5. 最近检测记录（最新10条）
6. 高危缺陷告警（置信度>0.9 或 面积>阈值）
7. 今日/本周新增统计

**优先级P2（可选实现）**：
8. 用户活跃度统计（仅管理员）
9. 检测耗时统计
10. 存储空间使用情况

## 4. 数据模型设计

### 4.1 数据库查询视图

**注意**：无需创建新表，所有统计数据通过查询现有表计算得出。

#### 现有数据表关系
```
users (用户表)
  ↓ user_id
bridges (桥梁表)
  ↓ bridge_id
defects (缺陷表)
  - defect_id
  - bridge_id (外键)
  - defect_type (缺陷类型)
  - confidence (置信度)
  - area (面积)
  - detected_at (检测时间)
  - deleted_at (软删除)
```

### 4.2 SQL查询模式

#### 模式1：基础统计（带权限过滤）
```sql
-- 用户概览统计
SELECT
    (SELECT COUNT(*) FROM bridges WHERE user_id = ? AND deleted_at IS NULL) AS bridge_count,
    (SELECT COUNT(*) FROM drones WHERE user_id = ? AND deleted_at IS NULL) AS drone_count,
    (SELECT COUNT(d.*) FROM defects d
     JOIN bridges b ON b.id = d.bridge_id
     WHERE b.user_id = ? AND d.deleted_at IS NULL) AS defect_count,
    (SELECT COUNT(DISTINCT d.image_path) FROM defects d
     JOIN bridges b ON b.id = d.bridge_id
     WHERE b.user_id = ? AND d.deleted_at IS NULL) AS detection_count;
```

#### 模式2：缺陷类型分布（聚合查询）
```sql
-- 缺陷类型统计
SELECT
    d.defect_type,
    COUNT(*) AS count,
    ROUND(AVG(d.confidence), 2) AS avg_confidence
FROM defects d
JOIN bridges b ON b.id = d.bridge_id
WHERE b.user_id = ? AND d.deleted_at IS NULL
GROUP BY d.defect_type
ORDER BY count DESC;
```

#### 模式3：时间序列趋势（GROUP BY日期）
```sql
-- 最近7天缺陷趋势
SELECT
    DATE(d.detected_at) AS date,
    COUNT(*) AS count
FROM defects d
JOIN bridges b ON b.id = d.bridge_id
WHERE b.user_id = ?
  AND d.deleted_at IS NULL
  AND d.detected_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY DATE(d.detected_at)
ORDER BY date ASC;
```

#### 模式4：桥梁排名（GROUP BY + ORDER BY）
```sql
-- 桥梁健康度排名（缺陷最多的前10座）
SELECT
    b.id AS bridge_id,
    b.bridge_name,
    COUNT(d.id) AS defect_count,
    SUM(CASE WHEN d.confidence >= 0.9 THEN 1 ELSE 0 END) AS high_risk_count,
    MAX(d.detected_at) AS last_detection
FROM bridges b
LEFT JOIN defects d ON d.bridge_id = b.id AND d.deleted_at IS NULL
WHERE b.user_id = ? AND b.deleted_at IS NULL
GROUP BY b.id, b.bridge_name
ORDER BY defect_count DESC
LIMIT 10;
```

// StatsServiceImpl 统计服务实现
type StatsServiceImpl struct {
    db *gorm.DB
}

// GetOverview 获取用户概览统计
func (s *StatsServiceImpl) GetOverview(currentUser *model.User) (*dto.StatsOverview, error) {
    var overview dto.StatsOverview

    // 1. 基础统计（子查询）
    if currentUser.IsAdmin() {
        // 管理员：全局统计
        s.db.Raw(`
            SELECT
                (SELECT COUNT(*) FROM bridges WHERE deleted_at IS NULL) AS bridge_count,
                (SELECT COUNT(*) FROM drones WHERE deleted_at IS NULL) AS drone_count,
                (SELECT COUNT(*) FROM defects WHERE deleted_at IS NULL) AS defect_count,
                (SELECT COUNT(DISTINCT image_path) FROM defects WHERE deleted_at IS NULL) AS detection_count
        `).Scan(&overview)
    } else {
        // 普通用户：仅自己的数据
        s.db.Raw(`
            SELECT
                (SELECT COUNT(*) FROM bridges WHERE user_id = ? AND deleted_at IS NULL) AS bridge_count,
                (SELECT COUNT(*) FROM drones WHERE user_id = ? AND deleted_at IS NULL) AS drone_count,
                (SELECT COUNT(d.id) FROM defects d
                 JOIN bridges b ON b.id = d.bridge_id
                 WHERE b.user_id = ? AND d.deleted_at IS NULL) AS defect_count,
                (SELECT COUNT(DISTINCT d.image_path) FROM defects d
                 JOIN bridges b ON b.id = d.bridge_id
                 WHERE b.user_id = ? AND d.deleted_at IS NULL) AS detection_count
        `, currentUser.ID, currentUser.ID, currentUser.ID, currentUser.ID).Scan(&overview)
    }

    // 2. 增量统计（今日、本周）
    var todayDefects, weekDefects, yesterdayDefects int

    baseQuery := s.db.Model(&model.Defect{})
    if !currentUser.IsAdmin() {
        baseQuery = baseQuery.Joins("JOIN bridges ON bridges.id = defects.bridge_id").
            Where("bridges.user_id = ?", currentUser.ID)
    }

    // 今日新增
    baseQuery.Where("DATE(defects.detected_at) = CURDATE()").Count(&todayDefects)
    overview.TodayDefects = todayDefects

    // 本周新增
    baseQuery.Where("defects.detected_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)").Count(&weekDefects)
    overview.WeekDefects = weekDefects

    // 昨日新增（计算趋势）
    baseQuery.Where("DATE(defects.detected_at) = DATE_SUB(CURDATE(), INTERVAL 1 DAY)").Count(&yesterdayDefects)

    // 3. 计算趋势
    if yesterdayDefects > 0 {
        overview.DefectTrend = float64(todayDefects-yesterdayDefects) / float64(yesterdayDefects) * 100
    }

    if todayDefects > yesterdayDefects {
        overview.TrendDirection = "up"
    } else if todayDefects < yesterdayDefects {
        overview.TrendDirection = "down"
    } else {
        overview.TrendDirection = "stable"
    }

    return &overview, nil
}

// 构建带权限的基础查询
func (s *StatsServiceImpl) buildAuthorizedQuery(currentUser *model.User) *gorm.DB {
    query := s.db.Model(&model.Defect{})

    if !currentUser.IsAdmin() {
        // 普通用户：JOIN桥梁表过滤
        query = query.Joins("JOIN bridges ON bridges.id = defects.bridge_id").
            Where("bridges.user_id = ?", currentUser.ID)
    }

    return query.Where("defects.deleted_at IS NULL")
}

// 使用示例
func (s *StatsServiceImpl) GetDefectTypeDistribution(currentUser *model.User, days int) (*dto.DefectTypeDistributionResponse, error) {
    query := s.buildAuthorizedQuery(currentUser)

    if days > 0 {
        query = query.Where("defects.detected_at >= DATE_SUB(CURDATE(), INTERVAL ? DAY)", days)
    }

    var distribution []dto.DefectTypeDistribution
    query.Select("defect_type, COUNT(*) AS count, ROUND(AVG(confidence), 2) AS avg_confidence").
        Group("defect_type").
        Order("count DESC").
        Scan(&distribution)

    // 计算百分比
    total := 0
    for _, d := range distribution {
        total += d.Count
    }
    for i := range distribution {
        distribution[i].Percentage = float64(distribution[i].Count) / float64(total) * 100
    }

    return &dto.DefectTypeDistributionResponse{
        Total:        total,
        Distribution: distribution,
    }, nil
}

### 13.1 健康评分算法详解

```go
// HealthScore = 100 - (缺陷总数 * 权重1 + 高危缺陷数 * 权重2)
func CalculateHealthScore(defectCount, highRiskCount int) float64 {
    const (
        WeightNormal   = 1.0   // 普通缺陷权重
        WeightHighRisk = 5.0   // 高危缺陷权重
    )

    score := 100.0 - float64(defectCount)*WeightNormal - float64(highRiskCount)*WeightHighRisk

    // 限制在[0, 100]范围
    if score < 0 {
        score = 0
    }

    return score
}

func GetHealthLevel(score float64) string {
    switch {
    case score >= 90:
        return "优秀"
    case score >= 70:
        return "良好"
    case score >= 50:
        return "一般"
    case score >= 30:
        return "较差"
    default:
        return "危险"
    }
}
```

```go
func DetermineSeverity(confidence float64, area float64) string {
    // 规则1: 置信度极高 + 面积大 = 紧急
    if confidence >= 0.95 && area >= 0.1 {
        return "紧急"
    }

    // 规则2: 置信度高 或 面积较大 = 严重
    if confidence >= 0.90 || area >= 0.05 {
        return "严重"
    }

    // 规则3: 置信度较高 或 面积一般 = 高危
    if confidence >= 0.85 || area >= 0.02 {
        return "高危"
    }

    return "一般"
}
```

### 13.3 时间粒度处理

```go
func GetDateFormat(granularity string) string {
    switch granularity {
    case "hour":
        return "%Y-%m-%d %H:00:00"
    case "day":
        return "%Y-%m-%d"
    case "week":
        return "%Y-%W"  // 年份-周数
    case "month":
        return "%Y-%m"
    default:
        return "%Y-%m-%d"
    }
}

func BuildTrendQuery(days int, granularity string) string {
    dateFormat := GetDateFormat(granularity)

    return fmt.Sprintf(`
        SELECT
            DATE_FORMAT(detected_at, '%s') AS date,
            COUNT(*) AS count
        FROM defects
        WHERE detected_at >= DATE_SUB(CURDATE(), INTERVAL %d DAY)
          AND deleted_at IS NULL
        GROUP BY DATE_FORMAT(detected_at, '%s')
        ORDER BY date ASC
    `, dateFormat, days, dateFormat)
}
```
