## 📋 测试脚本说明

本项目提供了两个测试脚本，用于验证系统功能：

## 🔍 统计模块测试覆盖

端到端测试会验证所有6个统计接口：

1. **GET /api/v1/stats/overview**
   - 概览统计（桥梁数、无人机数、缺陷总数等）

2. **GET /api/v1/stats/defect-types?days=30**
   - 缺陷类型分布（各类型缺陷数量、占比、平均置信度）

3. **GET /api/v1/stats/defect-trend?days=7&granularity=day**
   - 缺陷趋势统计（时间序列、峰值、日均值）

4. **GET /api/v1/stats/bridge-ranking?limit=10&order=worst**
   - 桥梁健康度排名（按缺陷数量排序）

5. **GET /api/v1/stats/recent-detections?limit=5**
   - 最近检测记录（最新检测的图片及缺陷数）

6. **GET /api/v1/stats/high-risk-alerts?limit=20**
   - 高危缺陷告警（高置信度或大面积缺陷）

7. **GET /api/v1/stats/high-risk-alerts?severity=urgent**
   - 紧急告警过滤（置信度≥0.95 且 面积≥0.1㎡）

## 🐛 常见问题

### Q1: 测试脚本提示 "jq: command not found"

**解决**: 安装 jq 工具
```bash
sudo apt-get install jq  # Ubuntu
brew install jq          # macOS
```

### Q2: 统计模块测试返回 401 错误

**原因**: 数据库中没有 `user1` 用户

**解决**: 运行端到端测试脚本，它会自动创建测试用户和数据

### Q3: 端到端测试执行到缺陷插入时失败

**原因**: MySQL 连接失败或权限不足

**解决**:
1. 检查 MySQL 是否运行：`sudo systemctl status mysql`
2. 检查密码是否正确：`mysql -uroot -p123456`
3. 确保数据库存在：`CREATE DATABASE IF NOT EXISTS bridge_detection;`

### Q4: 测试通过率不是 100%

**原因**: 可能是服务未启动或端口被占用

**解决**:
1. 确认服务正在运行：`curl http://localhost:8080/health`
2. 检查端口占用：`lsof -i :8080`
3. 查看服务日志

## 📝 自定义测试

### 修改测试参数

编辑脚本中的配置变量：

```bash
# doc/e2e_test.sh
BASE_URL="http://localhost:8080/api/v1"  # 修改服务地址
COOKIE_FILE="e2e_cookies.txt"            # 修改 Cookie 文件名
```

### 修改缺陷数据

编辑 SQL 插入脚本（第四部分）：

```sql
-- 增加更多缺陷
INSERT INTO defects (bridge_id, defect_type, ...) VALUES
($BRIDGE1_ID, '新缺陷类型', ...),
...
```

### 添加新的测试步骤

在脚本末尾添加：

```bash
# 步骤26：你的测试
print_step 26 $TOTAL_STEPS "测试描述"
YOUR_TEST=$(curl -s -b "$COOKIE_FILE" "${BASE_URL}/your-endpoint")
check_response "$YOUR_TEST" "200" "测试名称"
```
