## 1. 模块概述

### 1.1 功能范围

用户管理模块是桥梁缺陷检测系统的核心基础模块，负责处理所有用户相关的操作，包括：

**核心功能**:
- 用户注册 (自动成为普通用户)
- 用户登录 (支持用户名/邮箱登录)
- 用户登出
- 个人信息管理
- 用户列表查询 (管理员)
- 用户权限提升 (管理员)
- 用户删除 (管理员)

**用户角色**:
- **普通用户 (user/role=0)**: 可以管理自己的桥梁、无人机、检测任务
- **管理员 (admin/role=1)**: 拥有全部权限，可以管理所有用户和资源

### 1.2 技术栈

- **Web框架**: Gin
- **ORM**: GORM
- **数据库**: MySQL 8.0
- **认证方式**: Session + Cookie
- **密码加密**: bcrypt
- **配置管理**: YAML

### 1.3 设计原则

- **安全第一**: 密码必须加密存储，Session安全管理
- **权限分离**: 普通用户和管理员权限明确分离
- **数据隔离**: 用户只能访问自己的数据（管理员除外）
- **RESTful规范**: API设计遵循RESTful最佳实践
- **错误友好**: 提供清晰的错误信息，不暴露敏感数据

## 3. 数据模型设计

### 3.1 User 实体模型

```go
// internal/domain/model/user.go
package model

import (
    "time"
)

// User 用户实体
type User struct {
    ID        uint      `gorm:"primaryKey;autoIncrement" json:"id"`
    Username  string    `gorm:"type:varchar(50);uniqueIndex;not null" json:"username"`
    Password  string    `gorm:"type:varchar(255);not null" json:"-"` // 不序列化到JSON
    RealName  string    `gorm:"type:varchar(50);not null" json:"real_name"`
    Phone     string    `gorm:"type:varchar(20)" json:"phone"`
    Email     string    `gorm:"type:varchar(100);uniqueIndex;not null" json:"email"`
    Role      string    `gorm:"type:varchar(20);default:'user'" json:"role"` // "user" 或 "admin"
    CreatedAt time.Time `gorm:"autoCreateTime" json:"created_at"`
    UpdatedAt time.Time `gorm:"autoUpdateTime" json:"updated_at"`
}

// TableName 指定表名
func (User) TableName() string {
    return "users"
}

// IsAdmin 判断是否为管理员
func (u *User) IsAdmin() bool {
    return u.Role == "admin"
}
```

### 3.2 数据库表结构

```sql
CREATE TABLE `users` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(255) NOT NULL COMMENT '密码（bcrypt加密）',
  `real_name` varchar(50) NOT NULL COMMENT '真实姓名',
  `phone` varchar(20) DEFAULT NULL COMMENT '手机号',
  `email` varchar(100) NOT NULL COMMENT '邮箱',
  `role` varchar(20) NOT NULL DEFAULT 'user' COMMENT '角色: user/admin',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_username` (`username`),
  UNIQUE KEY `idx_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**索引说明**:
- `PRIMARY KEY (id)`: 主键索引
- `UNIQUE (username)`: 用户名唯一索引，用于登录和防重复
- `UNIQUE (email)`: 邮箱唯一索引，用于邮箱登录和防重复

### 3.3 字段约束

| 字段 | 类型 | 长度 | 必填 | 唯一 | 默认值 | 说明 |
|------|------|------|------|------|--------|------|
| id | bigint unsigned | - | 是 | 是 | AUTO_INCREMENT | 主键 |
| username | varchar | 50 | 是 | 是 | - | 3-50字符，字母数字下划线 |
| password | varchar | 255 | 是 | 否 | - | bcrypt加密，固定60字符 |
| real_name | varchar | 50 | 是 | 否 | - | 真实姓名 |
| phone | varchar | 20 | 否 | 否 | NULL | 手机号 |
| email | varchar | 100 | 是 | 是 | - | 邮箱格式验证 |
| role | varchar | 20 | 是 | 否 | 'user' | "user" 或 "admin" |
| created_at | datetime | - | 是 | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | datetime | - | 是 | 否 | CURRENT_TIMESTAMP | 更新时间 |

## 4. API接口设计

### 4.1 认证接口

#### 4.1.1 用户注册

**接口**: `POST /api/v1/auth/register`

**请求参数**:
```json
{
  "username": "user1",           // 必填，3-50字符
  "password": "password123",     // 必填，6-50字符
  "real_name": "张三",           // 必填，1-50字符
  "phone": "13800138000",        // 可选，11位数字
  "email": "user1@example.com"   // 必填，邮箱格式
}
```

**参数验证**:
- `username`: 3-50字符，只允许字母、数字、下划线，不能全数字
- `password`: 6-50字符
- `real_name`: 1-50字符
- `phone`: 11位数字或为空
- `email`: 标准邮箱格式

**响应成功** (200):
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "user_id": 1,
    "username": "user1",
    "role": "user"
  }
}
```

**响应失败**:
- 400: 参数验证失败 (用户名格式错误、密码过短等)
- 409: 用户名或邮箱已存在
- 500: 服务器内部错误

#### 4.1.2 用户登录

**接口**: `POST /api/v1/auth/login`

**请求参数**:

**登录规则**:
- `username` 字段支持用户名或邮箱

### 5.3 权限验证流程

#### 5.3.1 认证中间件 (AuthRequired)

```go
// internal/interfaces/http/middleware/auth.go
func AuthRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        session := sessions.Default(c)
        userID := session.Get("user_id")

        if userID == nil {
            response.Unauthorized(c)
            c.Abort()
            return
        }

        // 查询用户信息
        user, err := userService.GetByID(userID.(uint))
        if err != nil {
            response.Unauthorized(c)
            c.Abort()
            return
        }

        // 将用户信息存入上下文
        c.Set("current_user", user)
        c.Next()
    }
}
```

#### 5.3.2 管理员权限中间件 (AdminRequired)

```go
// internal/interfaces/http/middleware/permission.go
func AdminRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        user, exists := c.Get("current_user")
        if !exists {
            response.Forbidden(c)
            c.Abort()
            return
        }

        currentUser := user.(*model.User)
        if !currentUser.IsAdmin() {
            response.Forbidden(c)
            c.Abort()
            return
        }

        c.Next()
    }
}
```

#### 5.3.3 资源所有权验证

```go
// 用于验证用户是否有权访问资源
func CheckResourceOwnership(c *gin.Context, resourceUserID uint) bool {
    currentUser := c.MustGet("current_user").(*model.User)

    // 管理员可以访问所有资源
    if currentUser.IsAdmin() {
        return true
    }

    // 普通用户只能访问自己的资源
    return currentUser.ID == resourceUserID
}
```

## 6. 安全设计

### 6.1 密码安全

#### 6.1.1 密码加密

**使用 bcrypt 算法**:
- **算法**: bcrypt (基于Blowfish)
- **Cost**: 10 (默认，2^10 = 1024轮迭代)
- **Salt**: 自动生成，包含在hash中
- **输出长度**: 60字符

```go
import "golang.org/x/crypto/bcrypt"

// 加密密码
func HashPassword(password string) (string, error) {
    bytes, err := bcrypt.GenerateFromPassword(
        []byte(password),
        bcrypt.DefaultCost, // Cost = 10
    )
    return string(bytes), err
}

// 验证密码
func CheckPassword(password, hash string) bool {
    err := bcrypt.CompareHashAndPassword(
        []byte(hash),
        []byte(password),
    )
    return err == nil
}
```

**密码策略**:
- 最小长度: 6字符
- 最大长度: 50字符
- 建议: 包含大小写字母、数字、特殊字符
- 不允许: 纯数字、纯字母、与用户名相同

#### 6.1.2 密码存储

- **数据库字段**: `password` VARCHAR(255)
- **实际长度**: bcrypt输出固定60字符
- **JSON序列化**: 使用 `json:"-"` 标签，防止密码泄露

```go

    Password string `gorm:"type:varchar(255);not null" json:"-"`
    // ...
```

```go
func SetupSession(r *gin.Engine, config *config.Config) {
    store := cookie.NewStore([]byte(config.Session.Secret))

    store.Options(sessions.Options{

        Secure:   config.Session.Secure,   // HTTPS only
```

### 8.3 业务错误处理

```go
// 注册接口示例
func (h *UserHandler) Register(c *gin.Context) {
    var req RegisterRequest

    // 1. 参数绑定
    if err := c.ShouldBindJSON(&req); err != nil {
        response.BadRequest(c, "参数格式错误")
        return
    }

    // 2. 参数验证
    if err := ValidateRegisterRequest(req); err != nil {
        response.BadRequest(c, err.Error())
        return
    }

    // 3. 业务处理
    user, err := h.userService.Register(req)
    if err != nil {
        // 根据错误类型返回不同响应
        if errors.Is(err, ErrUserExists) {
            response.Conflict(c, "用户名或邮箱已存在")
            return
        }
        // 未知错误，记录日志但不暴露细节
        log.Error("注册失败", "error", err)
        response.InternalError(c)
        return
    }

    // 4. 成功响应
    response.Success(c, gin.H{
        "user_id":  user.ID,
        "username": user.Username,
        "role":     user.Role,
    })
}
```

```go
        response.BadRequest(c, "参数格式错误")
        return
    }

    user, err := h.registerUseCase.Execute(c.Request.Context(), req)
    if err != nil {

        return
    }

    response.Success(c, gin.H{
        "user_id":  user.ID,
        "username": user.Username,
        "role":     user.Role,
    })
}
```

#### Service模板

```go
// internal/domain/service/user_service.go
package service

import (
    "errors"
    "golang.org/x/crypto/bcrypt"
    "github.com/xinyiis/BridgeDefectDetectionSys/internal/domain/model"
    "github.com/xinyiis/BridgeDefectDetectionSys/internal/domain/repository"
)

type UserService struct {
    userRepo repository.IUserRepository
}

func NewUserService(userRepo repository.IUserRepository) *UserService {
    return &UserService{
        userRepo: userRepo,
    }
}

func (s *UserService) Register(username, password, realName, phone, email string) (*model.User, error) {
    // 1. 检查用户名是否存在
    existingUser, _ := s.userRepo.FindByUsername(username)
    if existingUser != nil {
        return nil, errors.New("用户名已存在")
    }

    // 2. 检查邮箱是否存在
    existingUser, _ = s.userRepo.FindByEmail(email)
    if existingUser != nil {
        return nil, errors.New("邮箱已存在")
    }

    // 3. 加密密码
    hashedPassword, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    if err != nil {
        return nil, err
    }

    // 4. 创建用户
    user := &model.User{
        Username: username,
        Password: string(hashedPassword),
        RealName: realName,
        Phone:    phone,
        Email:    email,
        Role:     0, // 普通用户
        Status:   1, // 启用
    }

    if err := s.userRepo.Create(user); err != nil {
        return nil, err
    }

    return user, nil
}
```
