量化需求迫切：仅“框出”病害不够，⼯程养护需要知道病害的具体⻓度、宽度，锈蚀

5.项⽬说明
【问题说明】
参赛团队需要针对⽆⼈机采集的桥梁（包括⾼架、跨江⼤桥、铁路桥等）⾼分辨率图
像，设计并实现⼀套深度学习算法系统。该算法系统需要解决以下核⼼问题：
多类别病害检测：精准识别混凝⼟裂缝、剥落/掉块、钢筋裸露、钢结构锈蚀、泛碱
（渗⽔）等常⻅病害。
抗⼲扰能⼒：能够有效区分真实的病害与污渍、阴影、接缝、爬墙虎等⼲扰物。
像素级分割：不仅要考虑⽬标检测，还需考虑实例分割，以便后续计算病害的⼏何参
数。
【⽤⼾期望】服务外包2
⽤⼾期望获得⼀个核⼼算法模块/系统，能够输⼊⽆⼈机原始图像，输出包含病害类
别、掩膜（Mask）及置信度的结果⽂件。
该算法应具备“快、准、细”的特点：在主流显卡上具备准实时处理能⼒；对微⼩病害
的召回率⾼；分割边缘贴合度⾼，能够⽀撑后续⽣成桥梁检测报告。
6.任务要求
【开发说明】
建议参赛团队完成以下核⼼任务：
（1）算法模型构建：选择并改进主流的计算机视觉模型（如YOLO系列、Mask R
CNN、Swin Transformer等），针对桥梁病害的⼩⽬标和⻓宽⽐极端（如细⻓裂缝）
特性进⾏⽹络结构优化。
（2）数据增强与处理：利⽤图像处理技术解决桥下光照不均、运动模糊等问题，通过
数据增强提升模型泛化能⼒。
（3）量化计算模块：在分割结果基础上，设计算法估算病害的物理⻓度和最⼤宽度
（需结合给定的像素-物理尺⼨转换系数）。
【技术要求与指标】
（1）算法开发环境：推荐使⽤ Python 3.8, PyTorch 1.8或TensorFlow 2.x。
（2）⽬标类别：需检测⾄少3类桥梁病害。
（3）性能指标：
以下指标只作为衡量模型在可参考数据集上的效果对⽐，但⽐赛评审打分不局限于这
些指标，还要看实际应⽤效果的展⽰情况。
①检测和分割精度：建议在参考测试数据集上的病害分类精度（mAP0.50.95）和
病害区域的交并⽐（IoU）不低于现有基于YOLO或Transformer等的基线模型。
②运⾏效率：在常规消费级GPU（如RTX4090）上，单张4K分辨率图像的推理耗时
不超过200ms。
