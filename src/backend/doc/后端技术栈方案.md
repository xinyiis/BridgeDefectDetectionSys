# 桥梁病害检测系统 - 后端技术栈方案（精简版）

## 项目定位

服务外包大赛参赛项目，**重点在算法展示**，后端只需完成基础功能支撑。

**设计原则：**
- 够用就好，快速开发
- 技术栈简单成熟
- 专注功能实现

---

## 一、功能需求详细分析

### 1.1 用户管理模块

#### 角色划分
- **普通用户（User）**：默认角色，只能操作自己的数据
- **管理员（Admin）**：可以查看和管理所有用户及数据

**1.1.1 用户注册**

- 支持用户名/邮箱 + 密码登录
- 需要区分普通用户和管理员登录
- 登录成功后创建Session会话

**1.1.3 用户登出**
- 清除Session会话
- 退出登录状态

**1.1.4 用户管理（仅管理员）**
- 查看所有用户列表和详细信息
- 提升普通用户为管理员权限

### 1.4 图片上传识别模块

#### 功能流程
**1.4.1 上传图片**
- 用户选择本地图片上传
- 支持常见图片格式（jpg、png等）
- 后端保存原始图片

**1.4.2 配置检测参数**
- 选择检测模型（不同的YOLO模型）
- 选择关联的桥梁（可选）
- 输入像素-实际尺寸转换系数（例如：1像素=5mm）

**1.4.3 调用算法检测**
- Go后端将图片发送到Python算法服务
- Python服务进行：
  - 图像预处理和增强
  - YOLO目标检测（识别缺陷位置和类别）
  - SAM实例分割（精细分割缺陷区域）
  - 计算缺陷尺寸（像素尺寸）

**1.4.4 返回检测结果**
- 显示标注后的结果图（带bbox和mask）
- 显示缺陷信息：
  - 缺陷类别（裂缝、剥落、锈蚀等）
  - bbox坐标
  - 置信度
  - 像素级尺寸（长度、宽度、面积）
  - 实际物理尺寸（根据转换系数计算）
- 展示处理耗时

**1.4.5 保存检测记录**
- 将检测结果保存到数据库
- 如果关联了桥梁，记录到该桥梁的历史缺陷信息

**1.4.6 智能分析（可选）**
- 用户点击"智能分析"按钮
- 将检测结果图和缺陷信息发送到大模型API
- 大模型返回缺陷状况分析报告（文本）

---

### 1.5 视频流识别模块

#### 功能流程
**1.5.1 配置检测任务**

**1.5.4 实时展示**

**1.5.5 保存缺陷记录**
- 检测到的缺陷自动保存到数据库
- 关联到选定的桥梁历史记录

**1.5.6 停止监测**
- 用户点击停止按钮

- 生成本次监测的汇总报告

**1.5.7 智能分析（可选）**

#### 4. 用户认证：简单Session
```go

```

**实现方式：**
- 登录后生成session存储用户ID和角色
- 中间件验证session
- 无需JWT、Redis等复杂方案

**示例：**
```go
// 登录
session := sessions.Default(c)
session.Set("user_id", user.ID)
session.Save()

// 中间件验证
func AuthRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        session := sessions.Default(c)
        if session.Get("user_id") == nil {
            c.JSON(401, gin.H{"error": "未登录"})
            c.Abort()
            return
        }
        c.Next()
    }
}
```

### 11.1 主函数

```go
// main.go
package main

import (
    "github.com/gin-gonic/gin"
    "github.com/gin-contrib/cors"
    "github.com/gin-contrib/sessions"
    "github.com/gin-contrib/sessions/cookie"
)

func main() {
    // 初始化数据库
    initDB()

    // 创建Gin路由
    r := gin.Default()

    // 跨域配置
    r.Use(cors.Default())

    // Session配置
    store := cookie.NewStore([]byte("secret"))
    r.Use(sessions.Sessions("mysession", store))

    // 静态文件（返回检测结果图片）
    r.Static("/uploads", "./uploads")

    // 路由分组
    api := r.Group("/api")
    {
        // 用户相关
        api.POST("/register", handleRegister)
        api.POST("/login", handleLogin)
        api.POST("/logout", handleLogout)

        // 需要认证的路由
        auth := api.Group("/")
        auth.Use(AuthRequired())
        {
            // 桥梁管理
            auth.GET("/bridges", getBridges)
            auth.POST("/bridges", createBridge)

            // 检测
            auth.POST("/detect/image", detectImage)
        }
    }

    r.Run(":8080")
}
```

### 11.2 数据模型
```go
// models/user.go
type User struct {
    ID        uint      `gorm:"primaryKey"`
    Username  string    `gorm:"unique;not null"`
    Password  string    `gorm:"not null"`
    Role      string    `gorm:"default:'user'"` // user/admin
    CreatedAt time.Time
}

// models/bridge.go
type Bridge struct {
    ID          uint      `gorm:"primaryKey"`
    Name        string    `gorm:"not null"`
    Location    string
    Description string
    UserID      uint      `gorm:"not null"`
    User        User      `gorm:"foreignKey:UserID"`
    CreatedAt   time.Time
}

// models/defect.go
type Defect struct {
    ID          uint      `gorm:"primaryKey"`
    BridgeID    uint      `gorm:"not null"`
    Bridge      Bridge    `gorm:"foreignKey:BridgeID"`
    DefectType  string    `gorm:"not null"` // 裂缝/剥落/锈蚀等
    ImagePath   string    `gorm:"not null"` // 原图路径
    ResultPath  string    `gorm:"not null"` // 结果图路径
    BBox        string    `gorm:"type:text"` // JSON格式
```

```go
// handlers/detection.go
func detectImage(c *gin.Context) {
    // 1. 接收上传的图片
    file, _ := c.FormFile("image")
    bridgeID := c.PostForm("bridge_id")
    pixelRatio := c.PostForm("pixel_ratio") // 像素转换系数

    // 2. 保存原图
    imagePath := fmt.Sprintf("./uploads/images/%s", file.Filename)
    c.SaveUploadedFile(file, imagePath)

    // 3. 调用Python服务检测
    result, err := callPythonDetect(imagePath)
    if err != nil {
        c.JSON(500, gin.H{"error": "检测失败"})
        return
    }

    // 4. 保存检测结果
    defect := Defect{
        BridgeID:   bridgeID,
        DefectType: result.Type,
        ImagePath:  imagePath,
        ResultPath: result.ResultImagePath,
        BBox:       result.BBox,
        Length:     result.Length * pixelRatio,
        Width:      result.Width * pixelRatio,
        Area:       result.Area * pixelRatio * pixelRatio,
        Confidence: result.Confidence,
    }
    db.Create(&defect)

    // 5. 返回结果
    c.JSON(200, gin.H{
        "defect_id": defect.ID,
        "result_image": result.ResultImagePath,
        "defects": result.Defects,
        "process_time": result.ProcessTime,
    })
}
```

### 11.4 调用Python服务
```go
// utils/python_client.go
type DetectionResult struct {
    Type            string
    ResultImagePath string
    BBox            string
    Length          float64
    Width           float64
    Area            float64
    Confidence      float64
    ProcessTime     float64
    Defects         []DefectInfo
}

func callPythonDetect(imagePath string) (*DetectionResult, error) {
    file, err := os.Open(imagePath)
    if err != nil {
        return nil, err
    }
    defer file.Close()

    // 构造multipart请求
    body := &bytes.Buffer{}
    writer := multipart.NewWriter(body)
    part, _ := writer.CreateFormFile("file", filepath.Base(imagePath))
    io.Copy(part, file)
    writer.Close()

    // 调用Python服务
    resp, err := http.Post(
        "http://localhost:8000/detect",
        writer.FormDataContentType(),
        body,
    )
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()

    // 解析响应
    var result DetectionResult
    json.NewDecoder(resp.Body).Decode(&result)

    return &result, nil
}
```

## 十三、技术选型总结

| 技术 | 作用 | 是否必需 |
|-----|------|---------|
| Gin | Web框架 | ✅ 必需 |
| MySQL + GORM | 数据库 | ✅ 必需 |
| Session | 用户认证 | ✅ 必需 |
| 本地文件系统 | 文件存储 | ✅ 必需 |
| HTTP调用 | Go↔Python | ✅ 必需 |
| CORS | 跨域处理 | ✅ 必需 |

**不需要的技术（过度设计）：**
- ❌ JWT、Casbin - Session够用
- ❌ Redis - 比赛项目不需要缓存
- ❌ MinIO/OSS - 本地存储够用
- ❌ RabbitMQ/Kafka - HTTP调用更简单
- ❌ Swagger、Zap、Viper - 快速开发不需要
