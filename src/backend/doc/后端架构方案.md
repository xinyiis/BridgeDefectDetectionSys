## 一、架构设计原则

### 1.1 核心原则

**SOLID 原则**：
- **S** - 单一职责原则：每个模块只负责一件事
- **O** - 开闭原则：对扩展开放，对修改关闭
- **L** - 里氏替换原则：子类可以替换父类
- **I** - 接口隔离原则：客户端不应依赖不需要的接口
- **D** - 依赖倒置原则：依赖抽象而非具体实现

**清晰分层**：
- 展示层（Handler）→ 业务层（Service）→ 数据层（Repository）
- 每层职责明确，降低耦合

**领域驱动设计（DDD）**：
- 以业务领域为核心
- 清晰的领域模型
- 聚合根管理

## 三、分层架构详解

### 3.1 架构分层

```
┌─────────────────────────────────────────────────────────┐
│                    Interfaces Layer                      │
│              （接口层 - HTTP/WebSocket）                  │
│   • Handler - 处理 HTTP 请求                             │
│   • Middleware - 认证、日志、CORS                        │
│   • Router - 路由配置                                    │
└────────────────────┬────────────────────────────────────┘
                     │ DTO
                     ↓
┌─────────────────────────────────────────────────────────┐
│                  Application Layer                       │
│                （应用层 - 业务流程）                       │
│   • UseCase - 用例编排                                   │
│   • DTO - 数据传输对象                                   │
│   • Mapper - DTO ↔ 领域模型转换                          │
└────────────────────┬────────────────────────────────────┘
                     │ 领域模型
                     ↓
┌─────────────────────────────────────────────────────────┐
│                    Domain Layer                          │
│                 （领域层 - 核心逻辑）                      │
│   • Model - 领域实体                                     │
│   • Repository Interface - 仓储接口                     │
│   • Domain Service - 领域服务                           │
└────────────────────┬────────────────────────────────────┘
                     │ 接口实现
                     ↓
┌─────────────────────────────────────────────────────────┐
│                Infrastructure Layer                      │
│               （基础设施层 - 技术实现）                    │
│   • Persistence - 数据持久化（MySQL/Redis）             │
│   • External - 外部服务（Python/AI API）                │
│   • FileSystem - 文件存储                                │
│   • WebSocket - 实时通信                                 │
└─────────────────────────────────────────────────────────┘
```

### 3.2 依赖方向

- **向内依赖**：外层依赖内层，内层不依赖外层
- **领域层**是核心，不依赖任何外层
- **依赖倒置**：通过接口解耦

```go
// internal/domain/model/user.go
package model

    ID        uint

    Email     string
    Password  string    // 加密后的密码
    Role      UserRole  // 枚举：User/Admin
    CreatedAt time.Time
    UpdatedAt time.Time

type UserRole string

const (
    RoleUser  UserRole = "user"
    RoleAdmin UserRole = "admin"
)

// 领域方法
func (u *User) IsAdmin() bool {
    return u.Role == RoleAdmin
}

func (u *User) CanManage(resource Resource) bool {
    if u.IsAdmin() {
        return true
    }
    return resource.GetOwnerID() == u.ID
}
```

### 4.3 领域服务（Domain Service）

```go
// internal/domain/service/auth_service.go
package service

type AuthService struct {
    userRepo repository.UserRepository
    jwtUtil  *utils.JWTUtil
}

func NewAuthService(userRepo repository.UserRepository, jwtUtil *utils.JWTUtil) *AuthService {
    return &AuthService{
        userRepo: userRepo,
        jwtUtil:  jwtUtil,
    }
}

func (s *AuthService) Login(ctx context.Context, username, password string) (string, error) {
    // 1. 查找用户
    user, err := s.userRepo.FindByUsername(ctx, username)
    if err != nil {
        return "", errors.ErrUserNotFound
    }

    // 2. 验证密码
    if !utils.ComparePassword(user.Password, password) {
        return "", errors.ErrInvalidPassword
    }

    // 3. 生成 Token
    token, err := s.jwtUtil.GenerateToken(user.ID, user.Role)
    if err != nil {
        return "", err
    }

    return token, nil
}
```

### 4.4 用例（UseCase）

```go
// internal/application/usecase/detection_usecase.go
package usecase

type DetectionUseCase struct {
    defectRepo     repository.DefectRepository
    bridgeRepo     repository.BridgeRepository
    pythonClient   *external.PythonClient
    storageService *filesystem.StorageService
}

func (uc *DetectionUseCase) DetectImage(
    ctx context.Context,
    req *dto.DetectionRequest,
) (*dto.DetectionResponse, error) {
    // 1. 验证桥梁权限
    bridge, err := uc.bridgeRepo.FindByID(ctx, req.BridgeID)
    if err != nil {
        return nil, errors.ErrBridgeNotFound
    }

    // 2. 保存上传图片
    imagePath, err := uc.storageService.SaveImage(req.ImageFile)
    if err != nil {
        return nil, err
    }

    // 3. 调用 Python 算法服务
    result, err := uc.pythonClient.Detect(ctx, imagePath, req.ModelType)
    if err != nil {
        return nil, err
    }

    // 4. 计算实际尺寸
    dimensions := calculateDimensions(result, req.PixelRatio)

    // 5. 保存缺陷记录
    defect := &model.Defect{
        BridgeID:    req.BridgeID,
        Type:        result.Type,
        ImagePath:   imagePath,
        ResultPath:  result.ResultPath,
        BoundingBox: result.BBox,
        Confidence:  result.Confidence,
        Dimensions:  dimensions,
        DetectedAt:  time.Now(),
    }
    if err := uc.defectRepo.Create(ctx, defect); err != nil {
        return nil, err
    }

    // 6. 返回响应
    return mapper.ToDetectionResponse(defect, result), nil
}
```

### 4.5 HTTP Handler

```go
// internal/interfaces/http/handler/detection_handler.go
package handler

type DetectionHandler struct {
    detectionUC *usecase.DetectionUseCase
    logger      logger.Logger
}

func (h *DetectionHandler) DetectImage(c *gin.Context) {
    // 1. 解析请求
    var req dto.DetectionRequest
    if err := c.ShouldBind(&req); err != nil {
        response.Error(c, errors.ErrInvalidRequest)
        return
    }

    // 2. 获取文件
    file, err := c.FormFile("image")
    if err != nil {
        response.Error(c, errors.ErrFileUploadFailed)
        return
    }
    req.ImageFile = file

    // 3. 获取当前用户
    userID := c.GetUint("user_id")
    req.UserID = userID

    // 4. 调用用例
    resp, err := h.detectionUC.DetectImage(c.Request.Context(), &req)
    if err != nil {
        h.logger.Error("检测失败", zap.Error(err))
        response.Error(c, err)
        return
    }

    // 5. 返回成功响应
    response.Success(c, resp)
}
```

```go
        Message: "success",
        Data:    data,
    })

        c.JSON(http.StatusOK, Response{

        return
    }

    // 未知错误
    c.JSON(http.StatusInternalServerError, Response{
        Code:    1000,
        Message: "内部服务器错误",
    })
}
```

### 12.2 缓存策略（可选）

```go
// 热点数据缓存
func (r *BridgeRepositoryImpl) FindByID(ctx context.Context, id uint) (*model.Bridge, error) {
    // 1. 尝试从缓存获取
    cacheKey := fmt.Sprintf("bridge:%d", id)
    if cached := r.cache.Get(cacheKey); cached != nil {
        return cached.(*model.Bridge), nil
    }

    // 2. 从数据库查询
    var bridge model.Bridge
    if err := r.db.First(&bridge, id).Error; err != nil {
        return nil, err
    }

    // 3. 写入缓存
    r.cache.Set(cacheKey, &bridge, 10*time.Minute)

    return &bridge, nil
}
```
